Evolver 数据文件格式

表面的初始构型从一个 ASCII 数据文件中读取。 参见 cube 或 mound 示例获取样例。 数据文件分为六个部分：
  • 定义和选项
  • 顶点
  • 边
  • 面
  • 体
  • 初始命令
通用数据文件主题：
  • 通用语法（适用于数据文件和命令）
  • 包含文件
  • 宏
在下面的语法描述中，关键字将使用大写。 constexpr 表示常量表达式， expr 表示任意表达式。n 或 k 表示整数， 如果用作有向元素标签，则可以带符号。 方括号表示可选项。'|' 表示"或"。

注意：通常公式可以出现在数字合法的任何位置（元素编号、约束编号等除外）。某些公式以公式形式存储，在每次使用时重新求值；例如约束、边界和量的公式。其他公式，如顶点坐标，在读取数据文件时求值，只存储数值。因此，如果用户将顶点坐标定义为 3*width，那么在运行时改变变量 width 的值不会影响该顶点。如果你不清楚某个位置适用哪种解释，请 dump 数据文件并查看该位置是转储了公式还是数字。

包含文件

可以使用标准 C 语言方法包含其他文件。 文件名必须用双引号括起来。如果文件不在当前目录中， 将搜索 EVOLVERPATH。 包含可以嵌套到 10 层。 示例：
#include  "common.stuff"

宏

宏是在解析之前将标识符替换为字符串的文本替换。宏仅在数据文件中定义，在命令提示符下不起作用。 简单宏（无参数）可以像 C 语言中那样定义： 
#DEFINE  identifier  string
identifier 必须是对解析器没有其他特殊含义的标识符。 string 是逻辑行的其余部分，不包括注释。 当 identifier 后续作为标记出现时，它将被替换为 string。 替换会重新扫描。不进行递归检查。 宏定义有最大长度限制（当前为 500 个字符）。注意：宏 标识符是独立的标记，因此如果 "-M" 翻译为 "-2"，这将被读作两个标记，而不是一个带符号的数字。 数据文件中的关键字 keep_macros 将使宏定义在运行时保持有效， 直到加载下一个数据文件。

变量定义

可以在数据文件顶部区域使用以下语法定义变量： 
PARAMETER identifier = constexpr
这将声明 identifier 为具有给定初始值的变量。 该值可以在运行时使用 A 命令或通过 赋值来更改。变量 可以在任何后续表达式或常量表达式中使用。 更改此处定义的变量会在 autorecalc 开启时 导致表面自动重新计算。 因此，只有在其他顶部区域声明中需要的变量才应在此处定义。

可以指定一个过程，在变量值每次更改时调用。数据文件顶部的语法为 

PARAMETER identifier = constexpr ON_ASSIGN_CALL procedure_name
此功能的目的是允许改变变量值时产生副作用。该值是一个无参数的过程，只能在数据文件的顶部区域赋值。但是，过程本身可以随时重新定义。示例（在数据文件顶部区域）： 
procedure tester();
parameter aaa = 4
parameter bbb = 5 on_assign_call tester
procedure tester() { aaa := bbb; }
注意，首先声明了 "tester"，这样 Evolver 在声明 "bbb" 时能将 "tester" 识别为过程名。但 "tester" 的完整声明可以位于数据文件的顶部区域或数据文件的 "read" 部分。

水平集约束声明

在数据文件顶部区域声明 水平集约束的格式为 
CONSTRAINT n [GLOBAL] [CONVEX] [NONNEGATIVE] [NONPOSITIVE] [NONWALL] [CONTENT_RANK n]
FORMULA FUNCTION  expr
[ENERGY:
 E1: expr
 E2: expr
 E3: expr]
[CONTENT:
 C1: expr
 C2: expr
 C3: expr]
可以使用 EQUATION 或 FUNCTION 作为 FORMULA 的同义词。 这定义了约束编号 n，其中 n 是正整数。 可选关键字 GLOBAL 表示约束自动应用于所有顶点（但不自动应用于边或面）。 GLOBAL 约束计入数量限制。如果指定 CONVEX， 则会在约束上的边附加一个额外的 间隙能量， 以防止它们试图短路凸边界。 NONWALL 表示此约束在顶点和边弹出时将被忽略， 各种 "star" 平方平均曲率方法不会将此约束视为镜像平面。 如果指定 NONNEGATIVE 或 NONPOSITIVE，则这是一个 单侧约束， 所有顶点将在每次迭代中被强制适当地满足约束。 FORMULA 表达式定义了实际约束的零水平集。 它可以写成方程形式，因为 '=' 被解析为低优先级的减号。 公式可以包含任何 Evolver 已知值的表达式， 给定特定顶点。最常用的只是顶点的坐标 (x,y,z)， 但可以使用变量、量值或顶点额外属性。 使用顶点额外属性是将一个公式自定义到各个顶点的好方法。 例如，如果有一个名为 zfix 的顶点额外属性， 可以使用一个约束 z = zfix 将顶点强制到各个 z 值， 当然要先为每个顶点分配正确的 zfix 值 （在细化或以其他方式创建顶点后确保修复 zfix）。 不要使用 '>' 或 '<' 表示不等式；使用 NONNEGATIVE 或 NONPOSITIVE。 C 语言中的条件表达式对于定义由多个表面平滑连接的约束非常有用， 例如带有半球形端盖的圆柱体。可以在表达式开头对变量赋值， 主要目的是在被积函数中仅对公共子表达式求值一次。 此类复合表达式的语法为 
variable :=  expr, expr
表达式的值是第二个表达式的值。

可选的 ENERGY 部分表示约束上的顶点或边被认为具有能量。 在 soapfilm 模型中， 后续行给出将沿约束上的每条边积分的向量场的分量。 在 string 模型中， 只需要一个分量，在约束上的每个顶点处求值。 这样做的主要目的是允许完全在约束上的面被省略。 它们本应具有的任何能量应在此处包含。 一种用途是在约束处获得规定的接触角。 此能量还应包括因省略的面而产生的 重力势能。 积分现在也在 fixed 边上求值， 这与 Evolver 早期版本不同。

可选的 CONTENT 部分表示约束上的顶点 （string 模型）或 边（soapfilm 模型）对体的面积或体积有贡献。 如果体边界在约束上的部分没有由面定义， 那么体体积必须从内容积分中获得贡献。 理解内容如何添加到体中以获得正确的符号是很重要的。 积分沿边的正方向求值。如果边在一个面上正向定向， 且该面在一个体上正向定向，则积分被添加到该体中。 这可能最终给出与你认为的自然方向相反的积分符号。 始终在加载新数据文件时检查以确保积分结果正确。 约束属性 content_rank 与内容积分配合使用。 如果一个顶点（string 模型）或一条边（soapfilm 模型）位于多个具有内容积分的约束上 （例如墙壁相交处），如果存在内容等级， 则等级最低的内容积分将贡献给负侧体的内容， 等级最高的内容积分将贡献给正侧体的内容。

参数化边界声明

可以在数据文件的顶部区域使用以下语法声明参数化边界： 
BOUNDARY n PARAMETERS k [CONVEX]
X1: expr
X2: expr
X3: expr
这定义了边界编号 n，其中 n 是正整数， k 是参数数量。如果指定 CONVEX， 则会在边界上的边附加一个额外的 间隙能量， 以防止它们试图短路凸边界。 后续行包含用参数 P1 以及可能的 P2、P3、... 表示的坐标函数。 参见 悬链线示例。 边界的能量和内容积分已实现， 语法与 水平集约束相同。

命名量

在数据文件中定义 命名量的语法为： 

 QUANTITY name   ENERGY|FIXED=value|CONSERVED|INFO_ONLY [LAGRANGE_MULTIPLIER constexpr]
[TOLERANCE constexpr]  [MODULUS constexpr] methodlist | FUNCTION methodexpr

这里 name 是用户分配的标识符，用于引用该量。 任何量必须声明为三种类型之一：

  • ENERGY 量被添加到表面的总能量中；
  • FIXED 量被约束到固定的目标值；
  • CONSERVED 量类似于 FIXED，运动被投影以保持该量守恒，但实际值不会被投影到给定值。
  • INFO_ONLY 量的值仅报告给用户。
对于固定量，可选的拉格朗日乘数值提供拉格朗日乘数的初始值（量的 "pressure" 属性）。 这是为转储文件设计的，因此在重新加载时无需进行迭代即可拥有有效的拉格朗日乘数。

对于固定量，容差属性用于判断收敛性。 当所有量偏差与容差的比率之和小于 1 时，表面被视为已收敛。 此求和还包括固定体积的体。如果容差未设置或为负， 则使用变量 target_tolerance 的值，其默认值为 0.0001。

每个量都有一个模量，它只是整个量的标量乘数。 模量为 0 将关闭能量量。默认模量为 1。 量定义的 methodlist 版本可以包含一个或多个 方法实例。 要包含先前明确定义的实例，请包含 METHOD instancename。 可以使用 GLOBAL_METHOD 代替 METHOD， 表示该方法适用于适当类型的所有元素；它等同于在方法定义中使用 GLOBAL。 要在量定义中实例化方法，你基本上是包含 实例定义，但没有实例名称。 一个量包含一个预定义方法实例和一个隐式定义实例的示例： 

method_instance qwerty method facet_scalar_integral
  scalar_integrand: x^2
quantity foobar energy method qwerty method edge_scalar_integral
  scalar_integrand: y^3

通常第二种隐式定义更方便。 可以在一个 methodlist 中包含多个方法实例 （当前上限为 50），它们的值相加后乘以量的模量得到量的值。 FUNCTION methodexpr 变体将量定义为先前定义的方法实例的函数。示例： 

method_instance qwerty method facet_scalar_integral
  scalar_integrand: x^2
quantity foobar energy function qwerty^3

非全局量可以通过在定义元素的数据文件行中添加量名称来逐个应用于元素。 它们也可以在运行时使用 set 和 unset 命令应用或取消应用。 可定向方法可以通过在数据文件中方法名称后加破折号来以负方向应用。 "set" 命令中的方向遵循元素生成时的方向。

应用于不同元素类型的方法可以在一个量中组合。 如果将这样的量应用于一个元素，那么该量的所有适当类型的方法实例都将应用于该元素。 量的原始附加关系会被记住， 因此如果将边方法应用于面，那么从该面细化创建的边将继承该边方法。

命名方法实例声明

方法实例 通常作为 命名量的 定义的一部分来定义， 但在某些情况下，量由多个方法实例组成，需要单独引用这些方法实例； 也许用户想知道各个实例的值。 在数据文件中定义命名方法实例的通用语法为： 
METHOD_INSTANCE name METHOD methodname [MODULUS constexpr]
 [ELEMENT_MODULUS attrname]  [GLOBAL] [parameters]
这里，name 是用户分配的名称，用于引用此特定实例。 methodname 是 Evolver 中的 预定义方法之一。 模量值乘以方法值得到实例值。默认模量为 1。 可以通过为元素类型指定一个 额外属性 attrname 来给各个元素赋予乘数；该属性必须先前已定义。 GLOBAL 使该方法适用于适当类型的所有元素。 非全局实例可以通过在定义元素的数据文件行中添加实例名称来逐个应用于元素。 它们也可以在运行时使用 set 和 unset 命令应用或取消应用。 可定向方法可以通过在名称后加破折号在数据文件中以负方向应用。 各个元素的方向可以在运行时使用 "set" 命令设置，例如 
set facet[123] orientation -1

每个方法可以有各种参数来针对实例进行特化。 当前指定的唯一参数是：

SCALAR_INTEGRAND: expr

其中 expr 是坐标的标量函数 （在 edge_general_integral 或 facet_general_integral 中也可以是切向量或法向量分量的函数）。 适当类型元素的元素属性也可以使用。

VECTOR_INTEGRAND:
Q1: expr
Q2: expr
Q3: expr

其中表达式是坐标的函数。 适当类型元素的元素属性也可以使用。

FORM_INTEGRAND:
Q1: expr
Q2: expr
Q3: expr
...

其中表达式是坐标的函数。 适当类型元素的元素属性也可以使用。当在 facet_2form_integral 方法中使用时， 形式分量按字典序排列， 即在 4D 中六个分量 12,13,14,23,24,34 将被列为 Q1 到 Q6。

PARAMETER_1 constexpr

用于向某些方法指定杂项数值参数。

K_FORM_ORDER constexpr

对于使用微分 k-形式的方法，这指定 k 的值。 当需要时，应出现在 FORM_INTEGRAND 之前。

表面维度

表面的默认维度为 2。如果不是，必须在数据文件的顶部区域声明。 对于一维表面（string 模型），只需包含一行 
STRING
默认维度 2 的 soapfilm 模型 等同于使用 
SOAPFILM
一般地，行 
SURFACE_DIMENSION n
定义表面具有维度 n。 超过 2 的维度仅在 simplex 模型中有效。 表面维度可以在运行时通过只读变量 surface_dimension 访问。

空间维度

空间的默认维度为 3。否则必须在数据文件的顶部区域声明，语法为 
SPACE_DIMENSION n
维度必须不超过 model.h 中 MAXCOORD 的值， 分发版本中为 4。 空间维度可以在运行时通过只读变量 space_dimension 访问。

额外属性声明

用户可以为元素定义 额外属性， 可以是单个值或最多八维数组。 如果这些属性要包含在数据文件中， 则数据文件的顶部区域必须包含适当的定义。 定义语法与 define 运行时命令使用的相同： 
DEFINE elementtype ATTRIBUTE name type [dim]...
其中 elementtype 是 vertex、edge、facet 或 body，name 是你选择的标识符，type 是 REAL 或 INTEGER（内部还有一个 ULONG 无符号长整型）， dim 是向量维度的可选表达式。 目前实数和整数类型之间没有实际区别，因为内部一切都存储为实数。 但将来可能会添加更多数据类型。 额外属性通过细分生成的同类型元素继承。 type 后面可以跟 FUNCTION， 再跟一个括号中的过程，在读取属性值时求值；在公式中， 可以使用 self 来引用相关元素以使用其属性， 特别是在某个点为属性赋值。 print 命令可用于以括号形式打印属性数组或数组切片。 示例： 
define edge attribute charlie real
define vertex attribute newx real[3]
define facet attribute knots real[5][5][5]
define edge attribute bbb real function { self.bbb := self.x+self.y }
警告：如果你打算在数据文件的顶部区域定义独立函数并将其放在属性声明之后，存在语法歧义。 你应该在属性之前定义独立函数，或者用其他类型的声明将它们分开。

二次模型声明

要声明数据文件以 二次模型列出表面， 数据文件的顶部区域应包含行 
QUADRATIC
对数据文件语法的唯一影响是 边部分可以列出边中点顶点。

重力常数声明

重力常数 的初始值可以在数据文件中使用以下语法设置 
GRAVITY_CONSTANT value
默认值为 1。

环面模型声明

要声明周期性边界条件（即将域设为平坦的 环面）， 在数据文件的顶部区域包含行 
TORUS
所有空间维度将是周期性的，周期向量在 周期声明中给出。 如果域被具有规定体积的体完全填充，则应使用行 
TORUS_FILLED
以防止退化的体积约束。

环面周期

如果使用周期性边界条件 （环面模型）， 基本单元平行六面体的周期向量可以在数据文件的顶部区域定义。 默认是单位立方体。语法是关键字 PERIODS 后跟每个周期向量的分量表达式： 
PERIODS
expr expr expr
expr expr expr
expr expr expr
此矩阵的大小取决于空间维度。 表达式中可以使用变量， 因此可以通过给变量赋新值来交互式地改变基本域。 每次更改此类变量时，请确保给出 recalc 命令， 以便重新计算周期矩阵。

视图矩阵

数据文件的顶部区域可以包含初始的 视图矩阵： 
VIEW_MATRIX
  constexpr constexpr constexpr constexpr
  constexpr constexpr constexpr constexpr
  constexpr constexpr constexpr constexpr
  constexpr constexpr constexpr constexpr
矩阵采用齐次坐标，平移分量在最后一列。 矩阵的大小比空间维度大一。 此矩阵将包含在所有转储文件中，因此视图可以在会话之间保存。 此矩阵由原生屏幕图形（'s' 命令）使用和设置， 仅适用于内部图形（Postscript、Xwindows 等）， 不适用于外部图形（geomview）。 元素可以在运行时通过 view_matrix[i][j] 读取或设置， 其中索引从 1 开始。 特别地，可以编写命令脚本来保存和重新加载特定的视图矩阵； 参见分发包中的 saveview.cmd。

视图变换

为了同时显示表面的多个变换， 可以在数据文件的顶部区域给出若干视图变换矩阵： 
VIEW_TRANSFORMS n
COLOR color
SWAP_COLORS
  constexpr constexpr constexpr constexpr
  constexpr constexpr constexpr constexpr
  constexpr constexpr constexpr constexpr
  constexpr constexpr constexpr constexpr
    ...
变换适用于所有图形（内部和外部）， 对于内部图形，变换优先于 视图矩阵。 恒等变换始终执行，因此不需要指定。 矩阵的数量跟在关键字 VIEW_TRANSFORMS 后面。 每个矩阵采用齐次坐标，平移分量在最后一列。 每个矩阵的大小比空间维度大一。 各个矩阵不需要特殊的分隔；Evolver 会持续读取表达式直到获取所需的所有数字。 每个矩阵前面可以有一个可选的 颜色， 适用于由该矩阵变换的面。颜色仅适用于一个变换； 它不会延续到下一个颜色规范。 如果存在 SWAP_COLORS，则在此矩阵应用时交换面的正面颜色和背面颜色。 变换可以通过 transforms 切换交互式地激活或停用。 内部变量 transform_count 记录变换的数量， 变换矩阵可以在运行时作为三维矩阵 view_transforms[][][] 访问。 视图变换生成器是控制视图变换的更高级方式。

函数和过程

通常独立的用户定义 函数 和 过程 在数据文件的 read 部分中定义， 但有时需要在数据文件的顶部区域定义它们， 以便在其他顶部区域声明中使用。 可以在顶部区域使用与 read 部分相同的语法定义它们。 注意，这适用于带参数传递的函数和过程， 用前导关键字 "function" 或 "procedure" 标记， 而不是像 "gg := {...}" 这样的命令定义。 警告：如果你打算在数据文件的顶部区域定义独立函数并将其放在属性声明之后，存在语法歧义。 你应该在属性之前定义独立函数，或者用其他类型的声明将它们分开。

优化参数

可以通过数据文件声明使变量在 迭代 或 hessian 命令期间接受优化： 
OPTIMIZING_PARAMETER identifier = constexpr PDELTA = constexpr PSCALE = constexpr
此类变量作为独立变量在优化过程中加入顶点坐标。 然而，它与坐标的不同之处在于关于它的梯度是数值计算的， 而不是解析计算的。因此它可以在允许使用变量的任何地方使用。 带优化参数的 Hessian 矩阵已实现。 可选的 pdelta 值是有限差分中使用的参数差异；默认值为 0.0001。 可选的 pscale 值是参数运动的乘数，用于对参数与表面能量进行"阻抗匹配"。 这些属性可以设置在任何参数上，以备作为优化参数使用。 在运行时，可以使用 FIX 和 UNFIX 命令切换参数是否优化。 即 fix radius 将使 radius 变量变为非优化（固定值）。 此外，pdelta 和 pscale 属性可以在运行时访问，如 
height.pscale := 2*height.pscale
在运行时，可以使用变量的 p_force 属性来查找约束校正前能量关于变量的变化率， 使用 p_velocity 属性来查找变量关于 'g' 命令缩放因子的变化率。 示例： 
g
print height.p_velocity

扩散

Evolver 可以模拟现实生活中相邻气泡之间的气体扩散现象。 这种扩散由穿过表面的压差驱动。 这通过数据文件第一部分中的关键字 DIFFUSION 调用， 后跟扩散常数的值，即内部变量 diffusion_coeff。 
DIFFUSION 0.1
迭代期间穿过面的扩散量计算为 scale*diffusion_coeff*facet_area*pressure_difference。 缩放因子作为迭代的时间步长包含在内。 该量被添加到或从面两侧的规定体积的体中减去。

如果你想对穿过各种表面的扩散速率进行更精细的控制， 可以在 string 模型中定义 edge_diffusion 边属性 或在 soapfilm 模型中定义 facet_diffusion 面属性， 并根据需要为边或面给出单独的值。 如果定义了该属性，则使用其值代替全局扩散常数。

元素列表

几何元素的列表遵循通用格式。每个元素在一行上定义。 行上的第一个条目是元素编号。编号不必连续，可以省略某些编号， 但请注意内部将按顺序重新编号。 数据文件中的原始编号可作为元素的 original 属性访问。 元素编号后面是基本定义数据，然后是任意顺序的可选属性。 除了下面列出的每种元素类型的特定属性外， 还可以为先前定义的任何 额外属性 指定值。语法是属性名称后跟适当数量的值。 还可以列出任意数量的 命名量或 方法实例。 这些将此元素的方法值添加到命名量中。 命名量或实例必须已在数据文件的顶部区域中声明。

顶点列表

数据文件顶点列表由行首关键字 VERTICES 开始。 后面是每行一个顶点规范的行。如果顶点不在 参数化边界上，语法为 
k   x y ... [FIXED] [CONSTRAINT c1 c2 ...]  [BARE]
            [quantityname ...] [methodname ...]
在 参数化边界上的顶点语法为 
k p1 [p2 ...]  BOUNDARY b [FIXED] [BARE] [quantityname ...]
   [methodname ...]
这里 k 是顶点编号，正整数。顶点 不需要按顺序列出，编号可以有间隔。 但是，如果不是连续顺序，则转储文件中的编号将不同。 x y ... 是坐标的常量表达式。 在参数化边界格式中，给出边界参数值而不是坐标。 如果给出 FIXED， 则顶点永远不会移动，可能的初始投影到约束除外。 如果给出 CONSTRAINT，则后面必须跟一个或多个 约束编号。 你可以列出任意多个约束， 只要任何时刻精确适用的约束是一致且独立的。 给定的坐标不必精确位于约束上；它们将被投影到约束上。 在参数化边界上的顶点不能同时在约束上。

BARE 属性只是对检查例程的指示， 表示此顶点在 soapfilm 模型中不应有相邻的面， 因此不会产生虚假警告。 当你想要显示裸线或勾勒基本域的轮廓时，这很有用。

可以列出任意数量的 命名量 或 方法实例。 这些将此元素的方法值添加到命名量中。 命名量或实例必须已在数据文件的顶部区域中 声明。

list vertices 命令 以数据文件格式打印顶点列表。

边列表

数据文件边列表跟在顶点列表后面， 由行首关键字 EDGES 开始。 后面是每行一个边规范的行，格式如下（此处行拼接）： 
k v1 v2 [midv] [s1 s2 s3] [WRAP w] [FIXED] [BOUNDARY b] \
   [CONSTRAINTS c1 [c2 ...]] [TENSION constexpr] [COLOR n] \
     [BARE]  [quantityname ...] [methodname ...]
这里 k 是边编号，编号规则与顶点相同。 v1 和 v2 是边的尾顶点和头顶点的编号。 在 二次模型中， 边中点可以作为第三个顶点 midv 列出 （否则将创建一个中点顶点）。 在 环面模型中， 后面必须跟符号 s1 s2 s3， 指示边如何环绕每个单元方向：+ 表示正向一次， * 表示无环绕，- 表示负向一次。 在非环面 对称群中， 每条边应有一个 WRAP 对称群元素编码为整数。 FIXED 表示 细分此边产生的所有顶点和边将具有 FIXED 属性； 它不意味着端点将自动被固定。 同样， BOUNDARY 和 CONSTRAINT 属性将由此边派生的所有边和顶点继承。 如果约束具有能量或内容被积函数，将为此边执行。 重要：如果约束编号为负，边能量和内容积分将按相反方向执行。 在 string 模型中， 默认张力为 1，在 soapfilm 模型中， 默认张力为 0。但是，可以在 soapfilm 模型中给边赋予非零张力， 它们将对能量有贡献。

如果 单纯形模型 生效，边比面低一个维度， 由有序顶点列表给出。 只有在具有积分的约束上的边才需要列出。

BARE 属性只是对检查例程的指示， 表示此边在 soapfilm 模型中不应有相邻的面， 因此不会产生虚假警告。 当你想要显示裸线或勾勒基本域的轮廓时，这很有用。

可以列出任意数量的 命名量 或 方法实例。 这些将此元素的方法值添加到命名量中。 命名量或实例必须已在数据文件的顶部区域中 声明。 如果量或实例具有方向相关的方法，名称后可以加破折号来反转应用的方向。

list edges 命令 以数据文件格式打印边列表。

面列表

数据文件面列表跟在边列表后面， 由行首关键字 FACES 开始。 后面是每行一个面规范的行，格式如下： 
k   e1 e2 ...  [FIXED] [TENSION constexpr] [BOUNDARY b] \
[CONSTRAINTS c1 [c2 ...]]     [NODISPLAY]   \
 [COLOR n]} [FRONTCOLOR n] [BACKCOLOR n] \
 [PHASE n] [quantityname ...] [methodname ...]
这里 k 是面编号，编号规则与 顶点相同。后面是有向边编号的列表， 按逆时针方向围绕面排列。 负的边编号表示与边列表中定义的相反方向。 最后一条边的头部必须是第一条边的尾部 （在 string 模型中使用技巧的情况除外）。 边的数量没有限制。 在 soapfilm 模型中， 如果面的边数超过三条，面将自动细分为三角形。 TENSION （同义词：DENSITY） 值是面的单位面积能量 （表面张力）； 默认值为 1。密度为 0 的面不施加力， 可用于定义体积或在显示中使用。 分数密度对于规定的接触角很有用。 NODISPLAY 防止面被显示。 COLOR 属性适用于面的两侧； FRONTCOLOR 适用于正面（边逆时针方向）， BACKCOLOR 适用于背面。 PHASE 编号在 string 模型中用于 确定不同相面之间边的表面张力（如果使用相）。

如果 单纯形模型 生效，边列表应替换为有向顶点编号列表。

可以列出任意数量的 命名量 或 方法实例。 这些将此元素的方法值添加到命名量中。 命名量或实例必须已在数据文件的顶部区域中 声明。 如果量或实例具有方向相关的方法，名称后可以加破折号来反转应用的方向。

面部分在 string 模型中是可选的。

list facets 命令 以数据文件格式打印面列表。

体列表

数据文件体列表跟在面列表后面， 由行首关键字 BODIES 开始。 后面是每行一个体规范的行，格式如下： 
k  f1 f2 f3 .... [VOLUME constexpr] [VOLCONST constexpr] [ACTUAL_VOLUME constexpr] \
[PRESSURE p] [DENSITY constexpr] [PHASE ]
这里 k 是体编号， f1 f2 f3 ... 是无序的有符号面编号列表。 正号表示面法向量（由面列表中边顺序的右手法则给出） 从体向外，负号表示法向量向内。 给出 VOLUME 值 constexpr 表示体具有 体积约束， 除非 理想气体模型 生效，此时 constexpr 是环境压力下的体积。 VOLCONST 是添加到体积的值；当面和边积分的体积计算 与真实体积相差一个常量时（如在 环面模型中可能发生）很有用。 ACTUAL_VOLUME 是一个数字， 可以在环面体积 volconst 计算给出错误答案的罕见情况下指定； volconst 将被调整以给出此体的体积。 给出 PRESSURE 值表示体被认为具有恒定的内部 压力； 这对于规定的平均曲率问题很有用。 它与规定体积不兼容。 给出 DENSITY 值表示 将包含 重力势能。

要在 string 模型中 赋予面 VOLUME、PRESSURE 或 DENSITY 属性， 只需用一个面定义一个体。

PHASE 编号在 soapfilm 模型中用于 确定不同相体之间面的 表面张力（如果使用相）。

BODIES 部分是可选的。

list bodies 命令 以数据文件格式打印体列表。

READ 部分

数据文件的最后部分可以包含命令。 这些命令被读取并立即执行，就像在命令提示符下输入的一样。 在数据文件中遇到关键字 READ 会导致 Evolver 从数据文件模式切换到命令模式， 并将数据文件的其余部分作为命令输入读取。 此功能对于通过细化、迭代、定义自己的命令等 自动初始化表面非常有用。 READ 部分是可选的。 示例： 
bodies
1   1 2 3 4 5 6 volume 1

read

// automatically do this when datafile is loaded
refine edge where on_constraint 1

// typical evolution
gogo := { g 5; r; g 10; r; g 20 }
list bottominfo 命令 打印转储文件中将打印的 READ 部分。
