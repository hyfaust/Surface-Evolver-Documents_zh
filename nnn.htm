命令语言

命令输入。当 Surface Evolver 处于提示 

Enter command:

时，它可以接受命令输入。命令输入有三种类型：主提示命令、图形模式命令和交互式响应。本页介绍主提示命令。主提示命令的关键字不区分大小写。命令在遇到换行符或分号时终止。换行符可以通过反斜杠 \ 转义。分号可以在引号字符串中转义。命令后的分号也会终止命令。如果命令行以反斜杠结尾，则下一行是当前命令的继续，而不是新命令的开始。如果命令语法需要数字表达式但未提供，则通常会提示输入该表达式。

命令分隔符。多条命令可以用分号分隔放在同一行上。分号将被忽略为命令名称的一部分。但是，如果一个表达式后面跟着一个分号，则该分号将终止该命令，而不是被忽略。用双括号括起来的注释可以放在命令中的任何地方。

  g 10; r; g 10    // 三次演化，中间有细化

复合命令。用花括号括起来的命令序列充当单个命令。此类序列可以出现在任何允许使用单个命令的地方。示例：

  { g 10; r; g 10; }
  foreach facet ff do { refine ff; ... }
  go := { g 10; r; g 10; }   // 定义命令

命令重复。命令可以在命令前面加上数字和左括号来重复给定次数。例如，命令 5( r; g 10 ) 将执行 5 次细化和 g 10 的序列。重复数字可以是表达式，并且可以放在括号之前或之后。重复可以嵌套。

赋值命令。请参阅下面的用户定义命令/过程/函数，了解命令定义语法。Evolver 2.90 新增功能：可以使用更传统的语法定义命令：

  commandname := { 命令序列 }

这也适用于函数和过程定义。

用户定义的命令/过程/函数

 用户可以定义自己的命令、过程和函数。命令只是文本替换；过程可以有参数；函数返回一个值。定义通过键入 define 后跟名称和定义来输入。定义以 # 作为一行上的第一个字符结束。除了注释之外，定义内部不允许使用 #。但是，定义中允许使用 define，因此可以嵌套。

命令定义

  define commandname
  命令 1
  命令 2
  ...
  #

然后通过键入命令名称来执行该命令。执行时，命令中的文本会替换定义中的文本，就像宏一样。对于带参数的命令，请使用下面的过程语法。在版本 2.96 中添加：为了兼容性，也可以使用 := 语法

  commandname := { 命令 1; 命令 2; ... }

过程定义

  define procname(arg1,arg2,...)
  命令 1
  命令 2
  ...
  #

调用过程时，文本替换用实际参数替换形式参数。

函数定义

  define real funcname(arg1,arg2,...)
  命令 1
  ...
  return expr
  #

函数在表达式中的使用方式与内置函数相同。函数使用 return 命令返回浮点值。字符串函数使用 define string 声明，并使用 return 返回字符串值。在版本 2.96 中添加：函数也可以使用简写定义

  funcname(arg1,...) := 表达式

其中表达式是一个单一表达式，可以使用参数。例如，

  sphere_area(r) := 4*pi*r^2

局部变量。函数和过程可以在其参数列表中声明局部变量。声明时用类型和名称（与参数声明相反，参数声明只有名称）。局部变量也可以在过程或函数体中使用 local 语句声明。变量名称是区分大小写的标识符。示例：

  define real test(x; real a,b; integer k)
  a := 3.14;
  b := x*a;
  k := floor(b);
  return k;
  #

当前显示的计算值和内存不会受局部变量值影响。

过程执行。过程通过键入其名称及其参数来执行，就像内置命令一样。函数可以在表达式中使用其名称及其参数来调用，就像内置函数一样。用户定义的名称优先于内置名称。用户定义的名称和内置名称都可以显示 list procedures 和 V 的文本。

用户定义命令的输出。在命令执行期间，可能会出现文本输出。默认情况下，它会发送到终端。但是，重定向允许将其发送到文件或通过管道传输到另一个命令。此外，如果命令以反引号括起来，则输出将作为字符串值捕获，以便在命令中使用，或分配给字符串变量。示例：

  greet := `printf "hello\n"`

所有用户命令、函数、过程、定义的变量和约束都可以使用 dump 命令保存到数据文件中，或通过 list procedures 命令在屏幕上列出。

变量赋值。变量可以通过 := 操作符赋值。示例：

  x := 3.5
  y := sin(x) + 2

变量可以是整数、实数或字符串类型。变量的类型由第一次赋值确定，之后不能更改。未使用 := 的赋值将被视为命令名称。

局部作用域。在 for、foreach 和 while 循环中定义的循环变量（使用 :=）仅在循环体中具有作用域。示例：

  for ( i := 1 ; i <= 10 ; i++ )
  { ... }

变量 i 仅在循环体内定义。

管道和重定向。命令输出可以通过 >> 重定向追加到文件，或通过 > 覆盖文件。示例：

  list vertices >> "vertices.txt"
  printf "value: %f\n", total_area > "area.txt"

输出也可以通过管道传输到系统命令，使用 |。示例：

  list vertices | sort

有关更多信息，请参阅 重定向。

控制结构

IF/THEN/ELSE。语法：

  IF ( expr ) command1

  IF ( expr ) command1 ELSE command2

括号是必需的。命令可以是复合命令（用花括号括起来）。else 部分是可选的。示例：

  if ( total_area > 100 ) { g 5; r; g 5; }
  if ( total_area > 100 ) g 5 else g 10

WHILE。语法：

  WHILE ( expr ) command

只要 expr 为真（非零），就重复 command。括号是必需的。示例：

  while ( total_area > 100 ) { g 1; r; }

DO/WHILE。语法：

  DO command WHILE ( expr )

类似于 while，但命令至少执行一次。注意：在 do 循环内，break 和 continue 行为与 C 类似，即 break 退出循环，continue 跳到循环的测试部分。

FOR。语法：

  FOR ( 初始化 ; 条件 ; 更新 ) command

这是 C 风格的 for 循环。三个部分都是可选的，但分号是必需的。如果省略条件，则视为真。示例：

  for ( i := 1 ; i <= 10 ; i++ ) { ... }
  for ( ;; ) { ... }  // 无限循环

FOREACH。语法：

  FOREACH elementtype name [ WHERE expr ] command

对满足可选条件的每个元素重复命令。name 是一个新变量，它接收当前元素的 id 号，并且可以在命令中使用，例如 name.id、name.valence 等。示例：

  foreach vertex v where on_constraint 1 do
  { ... }
  foreach facet ff where color == red do
  { set ff color blue; }

BREAK。在循环中使用，跳出最内层的 for、foreach、while 或 do 循环。

CONTINUE。在循环中使用，跳到最内层循环的下一次迭代。

RETURN。从用户定义的函数或过程返回。在函数中，return 后面必须跟一个表达式。

元素生成器

许多命令（如 set、delete、refine、fix 等）作用于元素集。指定元素集的表达式称为"生成器"。生成器可以是以下之一：

  • vertices、edges、facets、bodies、facetedges - 指定类型的所有元素
  • elementtype [ 表达式 ] - 单个元素，id 由表达式给出
  • elementtype name [ WHERE 表达式 ] - 满足表达式的所有元素（有关表达式的详细信息，请参阅 where 表达式）。where 是可选的。
  • elementtype [ 数字列表 ] - 指定 id 的元素列表
  • elementtype [ 数字 TO 数字 ] - id 在指定范围内的元素
  • name - 来自活动 foreach 循环的变量
  • set_of_elements - 以前保存的元素集

生成器可以附加限定条件，例如 where 子句或属性约束。生成器中的变量名称是区分大小写的。示例：

  refine edges where length > 0.5
  set facets where color == red density 0.5
  foreach vertex vv where on_constraint 1 do { ... }

ABORT

主提示命令。如果在脚本中运行，ABORT 命令会停止执行。在运行时使用 q 命令或 Ctrl-C 也能达到相同的效果，但在脚本中，ABORT 能确保在特定点停止。退出循环命令（break）也可以用于此目的。 
  abort

ADDLOAD

主提示命令。用于将另一个表面添加到当前表面。终止当前表面并加载新数据文件。 
  ADDLOAD filenamestring
filenamestring 是数据文件的名称，可以是带引号的字符串或字符串变量。这不会重新初始化任何内容，而是将新文件中的元素添加到当前表面中。适用于脚本中需要组合多个表面的场景。 有关加载新表面并重新初始化的信息，请参阅 load。 有关在不完全重新初始化的情况下替换表面的信息，请参阅 replace_load。 通配符匹配在某些系统上有效（Windows、Linux，可能还有其他系统），但使用通配符时要非常小心，因为可能会出现意外的匹配。

AREAWEED

主提示命令。删除面积小于给定值的面片。 
  AREAWEED expr
删除面积小于 expr 的面片。可用于清理演化过程中产生的小面积面片。另请参阅 edgeweed。

BINARY_PRINTF

主提示命令。用于以二进制格式输出数据。 
  BINARY_PRINTF string,expr,expr,...
使用二进制格式写入文件。格式字符串指定输出格式。这与 printf 类似，但输出为二进制格式。

BODY_METIS

主提示命令。使用 METIS 库对体进行分区。 
  BODY_METIS n
将体集划分为 n 个部分，使用 Karypis 和 Kumar 的 METIS 库（如果该库已编译到 Evolver 中）。 分区号存储在体的额外属性 bpart 中（如果尚不存在，则会创建该属性）。 示例： 
  body_metis 20;
  set body color (bpart imod 15) + 1;
有关对面片进行分区的信息，请参阅 metis。

BREAKPOINT

主提示命令。在用户定义的命令中设置断点。 
  BREAKPOINT
当 Evolver 遇到 breakpoint 命令时，它会暂停执行并返回到主提示。然后用户可以检查变量、执行命令等。使用 whereami 命令可查看导致断点的命令堆栈。使用 q 命令可从断点恢复执行。

CHDIR

主提示命令。更改当前工作目录。 
  CHDIR stringexpr
更改 Evolver 的当前工作目录。这对于加载和保存文件很有用。 在某些系统上，也可以使用 system 命令来更改目录。

CLOSE_SHOW

主提示命令。关闭图形显示窗口。 
  CLOSE_SHOW
关闭当前图形显示窗口。这与在图形模式下键入 q 的效果相同。

CONVERT_TO_QUANTITIES

主提示命令。将表面从旧式的固定约束转换为使用命名量和方法实例。 
  CONVERT_TO_QUANTITIES
此命令尝试自动将旧式约束转换为命名量。这对于将旧数据文件更新为新格式很有用。

DEFINE

主提示命令。用于定义命令、过程、函数、变量、数组和属性。语法： 
  DEFINE 命令名        // 定义命令
  DEFINE procname(arg1,...)  // 定义过程
  DEFINE REAL funcname(arg1,...)  // 定义函数
  DEFINE varname := 表达式  // 定义变量
  DEFINE arrayname 类型[大小]  // 定义数组
  DEFINE elementtype ATTRIBUTE name 类型  // 定义额外属性
有关更多信息，请参阅 命令定义。

DELETE

主提示命令。删除元素集。 
  DELETE generator
删除由生成器指定的元素。这与 dissolve 类似，但 delete 也会删除连接的低维元素（如果它们变得未使用）。例如，删除边会删除其端点（如果它们没有其他连接）。另请参阅 dissolve。 示例： 
  delete edges where length < 0.001
  delete facets where color == green

DELETE_TEXT

主提示命令。删除文本显示。 
  DELETE_TEXT expr
删除编号为 expr 的文本显示。有关文本显示的信息，请参阅 display_text。

DETORUS

主提示命令。将环面模型中的表面转换为普通表面。 
  DETORUS
将环面模型中的表面"展开"为普通欧几里得空间中的表面。这对于可视化环面中的表面很有用。

DIRICHLET

主提示命令。执行一步 Dirichlet 能量最小化。 
  DIRICHLET
使用 Dirichlet 能量（梯度的平方积分）代替面积/体积进行一步演化。这对于参数化表面很有用。

DIRICHLET_SEEK

主提示命令。沿最速下降方向执行能量最小化搜索。 
  DIRICHLET_SEEK
沿最速下降方向搜索 Dirichlet 能量的最小值。

DISPLAY_TEXT

主提示命令。在图形显示上显示文本。 
  DISPLAY_TEXT stringexpr
在图形窗口中显示文本字符串。文本将显示在屏幕上，直到使用 delete_text 命令删除。 每个文本显示都有一个编号，可用于删除。返回值是文本编号。

DISSOLVE

主提示命令。溶解（删除）元素集。 
  DISSOLVE generator
溶解由生成器指定的元素。与 delete 不同，dissolve 不会删除连接的低维元素。例如，溶解面片会留下其边和顶点。溶解边会合并其端点。 示例： 
  dissolve vertices where valence == 0
  dissolve edges where valence == 1

DUMP

主提示命令。将当前表面保存到数据文件。 
  DUMP stringexpr
将当前表面保存到由 stringexpr 指定的文件。文件格式为 Evolver 数据文件格式。这可用于保存中间结果或创建检查点。 示例： 
  dump "checkpoint.dmp"

DUMP_MEMLIST

主提示命令。转储内存分配列表以供调试。 
  DUMP_MEMLIST
将所有当前内存分配写入文件 memlist。这对于调试内存泄漏很有用。

EDGE_MERGE

主提示命令。合并边。 
  EDGE_MERGE expr
合并两个共享顶点的边，如果它们之间的角度小于 expr（以度为单位）。这对于简化表面很有用。

EDGESWAP

主提示命令。交换边。 
  EDGESWAP generator
交换由生成器指定的边的对角线。在三角形网格中，这相当于将四边形的对角线从一条对角线切换到另一条对角线。这对于改善网格质量很有用。 条件：
  • 边必须恰好有两个相邻的面片。
  • 两个面片必须形成一个凸四边形。
  • 交换后，新边必须比旧边短。
如果边不满足这些条件，edgeswap 将跳过它。 示例： 
  edgeswap edges where length > 0.5
另请参阅 t1_edgeswap。

EDGEWEED

主提示命令。删除长度小于给定值的边。 
  EDGEWEED expr
删除长度小于 expr 的边。对于固定顶点，不做任何操作。 示例： 
  edgeweed 0.01

EIGENPROBE

主提示命令。探测 Hessian 矩阵的特征值。 
  EIGENPROBE expr
使用逆迭代法寻找最接近 expr 的特征值和特征向量。此命令需要使用线性元素或完全二次元素，并且需要具有二次能量或线性约束（否则，Hessian 没有实际意义）。该命令使用迭代法，因此需要一个初始解。Hessian 为零的每个子空间使用迭代。 结果仅在所选子空间内为真。 示例： 
  eigenprobe 0
  eigenprobe 0.1

EPRINT

主提示命令。打印表达式及其值。 
  EPRINT expr
打印表达式及其值。通常用于调试命令定义，以查看表达式求值的中间结果。

EQUIANGULATE

主提示命令。对三角形进行等角化细分。 
  EQUIANGULATE
对网格进行操作，通过边翻转来优化三角形网格。此命令尝试最小化三角形对之间的最大角度。对于给定的一对共享一条边的三角形，equiangulate 会检查是否可以通过切换对角线来减小最大角度，如果可以则进行切换。 这是提高三角形网格质量的一个重要操作。另请参阅 refine。

ERRPRINTF

主提示命令。向错误输出打印格式化文本。 
  ERRPRINTF formatstring,args...
与 printf 相同，但输出到错误输出流 stderr 而不是 stdout。在 Unix 系统上，这允许输出重定向，因为 stdout 通常用于图形输出。

EXEC

主提示命令。执行命令文件。 
  EXEC filenamestring
执行由 filenamestring 指定的文件中的命令。命令以文本行的形式读取，就像在终端上输入它们一样。 这与 read 命令类似，但 exec 命令使用完整的 Evolver 命令语法。注意：对于大多数用途，应使用 read 命令代替。区别在于 exec 可以读取完整的 Evolver 命令。

EXPRINT

主提示命令。打印表达式及其值。 
  EXPRINT expr
打印表达式及其值。这与 eprint 类似，但更通用，可以打印完整表达式的值，包括字符串。

FACET_CROSSCUT

主提示命令。用一条线切割面片。 
  FACET_CROSSCUT f_id,x1,y1,z1,x2,y2,z2
在面片 f_id 上用 (x1,y1,z1) 到 (x2,y2,z2) 定义的线进行切割。该面片将被切割成两部分，并且会创建新的边和顶点。

FACET_MERGE

主提示命令。合并面片。 
  FACET_MERGE expr
合并共享边的两个面片，如果它们之间的角度小于 expr（以度为单位）。 条件：
  • 两个面片必须共享一条边。
  • 合并后形成的面片必须是平面的（如果不在平面公差内，则会警告用户）。
如果边不满足这些条件，facet_merge 将跳过它。

FIX

主提示命令。固定元素集。 
  FIX generator
固定由生成器指定的元素。固定元素在演化期间不会移动。 示例： 
  fix vertices where on_constraint 1
  fix edge[5]
另请参阅 unfix。

FLUSH_COUNTS

主提示命令。刷新操作计数器。 
  FLUSH_COUNTS
重置所有内部计数器（如细化计数、边翻转计数等），并将它们保存到 logfile。此命令在每个演化步骤中自动调用。

FREE_DISCARDS

主提示命令。释放已删除元素占用的内存。 
  FREE_DISCARDS
释放由已删除元素（如通过 delete 或 dissolve 删除的元素）占用的内存。这也可以释放被覆盖的额外属性值。此命令在自动重编号期间自动调用。

GEOMPIPE

主提示命令。设置图形管道。 
  GEOMPIPE stringexpr
将图形输出通过管道传输到外部程序。用于指定要发送几何输出的外部程序的命令行。 示例： 
  geompip "geomview -c -"

GEOMVIEW

主提示命令。启动 Geomview 图形显示。 
  GEOMVIEW stringexpr
启动 Geomview 图形显示程序。如果提供了参数，则将其用作 Geomview 的命令行选项。 示例： 
  geomview
  geomview "-c -wpos 100,100"

HELP

主提示命令。提供帮助信息。 
  HELP
  HELP topic
如果单独使用，则显示可用帮助主题的列表。如果使用参数，则显示有关指定主题的信息。可以使用 Evolver 中的本地帮助数据库，也可以使用在线帮助。可以通过 帮助语言命令 选择语言。

HESSIAN

主提示命令。执行一步 Hessian 能量最小化。 
  HESSIAN
执行一步 Hessian 能量最小化。这相当于执行 hessian_menu 然后执行 s 命令。对于二次能量，这给出精确解。对于线性边和线性约束，这是一个牛顿步骤。对于非线性元素，这近似于牛顿法，并且可能不收敛。 对于非线性元素，请改用 optimize。 此命令需要线性或二次元素以及线性约束。 另请参阅 hessian_seek、hessian_menu。

HESSIAN_MENU

主提示命令。进入 Hessian 菜单。 
  HESSIAN_MENU
进入 Hessian 菜单。此菜单提供多种与 Hessian 相关的操作。要直接执行 Hessian 步骤，请使用 hessian 命令。

HESSIAN_SEEK

主提示命令。沿最速下降方向执行 Hessian 能量最小化搜索。 
  HESSIAN_SEEK
  HESSIAN_SEEK expr
沿最速下降方向搜索 Hessian 能量的最小值。如果提供了参数 expr，则将其用作搜索方向的倍数。 这对于确保 Hessian 步骤不会"过冲"到更高能量点很有用。 另请参阅 hessian、hessian_menu。

HISTOGRAM

主提示命令。生成元素属性的直方图。 
  HISTOGRAM generator,expr
生成指定元素属性值的直方图。第一个参数是元素生成器，第二个参数是每个元素的表达式。 示例： 
  histogram facets where color == red, area
  histogram edges, length

HISTORY

主提示命令。显示命令历史记录。 
  HISTORY
显示之前输入的命令的历史记录。

LAGRANGE

主提示命令。将边设置为 Lagrange 模式。 
  LAGRANGE expr
将边元素设置为 Lagrange 有限元模式。参数 expr 是多项式的阶数（必须 >= 2）。这为更高阶的有限元建模提供了支持。 示例： 
  lagrange 2
  lagrange 3
另请参阅 quadratic、linear。

LANCZOS

主提示命令。使用 Lanczos 方法计算 Hessian 矩阵的特征值。 
  LANCZOS
  LANCZOS expr
使用 Lanczos 方法计算 Hessian 矩阵的特征值。默认情况下，它尝试找到所有负特征值。如果提供了参数，则将其用作特征值的数量上限。 此命令需要使用二次或 Lagrange 元素，并且需要线性约束。 输出包括找到的特征值列表以及每个特征值的特征向量范数。

LINEAR

主提示命令。将边设置为线性模式。 
  LINEAR
将边元素设置为线性模式。这意味着边在顶点之间是直线，表面由三角形组成。这是默认模式。 另请参阅 quadratic、lagrange。

LIST

主提示命令。列出元素。 
  LIST generator
列出由生成器指定的元素。有关详细输出，请参阅 show。 示例： 
  list vertices
  list edges where on_constraint 1
  list facets where color == red
  list bodies

LIST ATTRIBUTES

列出额外属性。 
  LIST ATTRIBUTES
列出所有额外属性的定义。

LIST BOTTOMINFO

列出底部信息。 
  LIST BOTTOMINFO
列出数据文件中的底部信息。

LIST PROCEDURES

列出用户定义的过程和函数。 
  LIST PROCEDURES
列出所有用户定义的过程、函数和命令。

LIST TOPINFO

列出顶部信息。 
  LIST TOPINFO
列出数据文件中的顶部信息。

LOAD

主提示命令。加载数据文件。 
  LOAD filenamestring
加载由 filenamestring 指定的 Evolver 数据文件。这会重新初始化所有内容，包括删除当前表面。如果想要在不重新初始化的情况下添加表面，请使用 addload。 通配符匹配在某些系统上有效（Windows、Linux，可能还有其他系统），但使用通配符时要非常小心，因为可能会出现意外的匹配。

LOGFILE

主提示命令。设置日志文件。 
  LOGFILE stringexpr
设置日志文件的名称。日志文件记录演化过程中的各种统计数据（如面积、能量等）。每个演化步骤都会在日志文件中添加一行。通过使用空字符串，可以关闭日志记录。 示例： 
  logfile "evolution.log"
  logfile ""  // 关闭日志
另请参阅 keylogfile。

KEYLOGFILE

主提示命令。设置按键日志文件。 
  KEYLOGFILE stringexpr
设置按键日志文件的名称。按键日志文件记录用户输入的所有命令。这对于记录会话很有用。 示例： 
  keylogfile "keystrokes.log"
  keylogfile ""  // 关闭日志

LONGJ

主提示命令。执行长跳转。 
  LONGJ expr
执行"长跳转"到最近的局部最小值。expr 是跳转的大小倍数。这与 saddle 命令不同，它不是在鞍点方向跳转，而是沿能量下降最快的方向跳转更大的距离。

MATRIX_INVERSE

主提示命令。计算矩阵的逆。 
  MATRIX_INVERSE var,expr,expr,...
计算矩阵的逆。第一个参数是结果变量，后面是矩阵元素（按行排列）。矩阵必须是方阵。结果存储在指定的变量中。 示例： 
  matrix_inverse result, 1, 2, 3, 4

MATRIX_MULTIPLY

主提示命令。计算矩阵乘法。 
  MATRIX_MULTIPLY var,expr,expr,...
计算矩阵乘法。第一个参数是结果变量，后面是两个矩阵的元素。结果存储在指定的变量中。

METIS

主提示命令。使用 METIS 对面片进行分区。 
  METIS n
将面片集划分为 n 个部分，使用 Karypis 和 Kumar 的 METIS 库。 分区号存储在面片的额外属性 fpart 中（如果尚不存在，则会创建该属性）。 示例： 
  metis 20;
  set facet color (fpart imod 15) + 1;
有关对体进行分区的信息，请参阅 body_metis。

MOVE

主提示命令。移动顶点。 
  MOVE generator dx,dy,[dz]
将由生成器指定的顶点移动指定的距离。 示例： 
  move vertex[1] 0.5, 0.3
  move vertices where on_constraint 1 0.1, 0.1, 0.1

NEW_VERTEX

主提示命令。创建一个新顶点。 
  NEW_VERTEX x,y,[z]
创建一个具有指定坐标的新顶点。新顶点的 id 为 max_vertex_id + 1（或如果使用自动重编号，则为下一个可用 id）。返回值是新顶点的 id。 示例： 
  v := new_vertex 1.0, 2.0, 3.0
注意：通常最好在数据文件中定义顶点，但 new_vertex 在编写创建新几何体的脚本时很有用。

NEW_EDGE

主提示命令。创建一个新边。 
  NEW_EDGE v1_id,v2_id
创建一个连接两个现有顶点的新边。新边的 id 为 max_edge_id + 1。返回值是新边的 id。 示例： 
  e := new_edge 1, 2
注意：通常最好在数据文件中定义边，但 new_edge 在编写创建新几何体的脚本时很有用。

NEW_FACET

主提示命令。创建一个新面片。 
  NEW_FACET e1_id,e2_id,e3_id,...
创建一个具有指定边的新面片。新面片的 id 为 max_facet_id + 1。返回值是新面片的 id。 示例： 
  f := new_facet 1, 2, 3

NEW_BODY

主提示命令。创建一个新体。 
  NEW_BODY
  NEW_BODY f1_id,f2_id,...
创建一个新体。如果提供了参数，则将指定的面片添加到新体中。新体的 id 为 max_body_id + 1。返回值是新体的 id。 示例： 
  b := new_body 1, 2, 3, 4

NOTCH

主提示命令。执行凹口操作。 
  NOTCH expr
在表面的凹口处执行操作。此命令主要用于调试和研究。

OMETIS

主提示命令。使用有序 METIS 对面片进行分区。 
  OMETIS n
使用有序 METIS 对面片集进行分区。这与 metis 类似，但保持面片的顺序。分区号存储在面片的额外属性 fpart 中。

OOGLFILE

主提示命令。设置 OOGL 输出文件。 
  OOGLFILE stringexpr
设置 OOGL 格式输出文件的名称。OOGL（Object-Oriented Graphics Library）格式用于 Geomview。文件名 "" 表示关闭输出。 示例： 
  ooglfile "surface.oogl"
  ooglfile ""  // 关闭输出

OPTIMIZE

主提示命令。执行一步能量最小化优化。 
  OPTIMIZE
使用共轭梯度法执行一步能量最小化。此命令适用于非线性元素和约束，而 hessian 仅适用于二次能量和线性约束。对于二次情况，hessian 更快。 示例： 
  g 10; optimize; g 10

PAUSE

主提示命令。暂停执行。 
  PAUSE
暂停执行并返回到主提示。这对于调试脚本很有用。使用 q 命令可恢复执行。

PERMLOAD

主提示命令。永久加载数据文件。 
  PERMLOAD filenamestring
加载由 filenamestring 指定的数据文件，但不重新初始化任何内容。这与 load 不同，后者会重新初始化。这主要用于高级用户和开发人员。

POP

主提示命令。弹出（删除）三角形的边。 
  POP generator
尝试弹出（删除）由生成器指定的三角形边。这是对三角形网格的一种基本操作，可以将三角形退化为边。 另请参阅 pop_edge_to_tri、pop_tri_to_edge。

POP_EDGE_TO_TRI

主提示命令。将边弹出为三角形。 
  POP_EDGE_TO_TRI generator
尝试将由生成器指定的边"弹出"为三角形。这会将一条边替换为一个三角形（三个边和一个面片）。

POP_QUAD_TO_QUAD

主提示命令。将四边形弹出为四边形。 
  POP_QUAD_TO_QUAD generator
尝试将四边形（四个边的面片）弹出为另一个四边形。这是对四边形网格的一种基本操作。

POP_TRI_TO_EDGE

主提示命令。将三角形弹出为边。 
  POP_TRI_TO_EDGE generator
尝试将三角形（三个边的面片）弹出为一条边。这是 pop_edge_to_tri 的逆操作。

POSTSCRIPT

主提示命令。生成 PostScript 输出。 
  POSTSCRIPT filenamestring
将当前表面的 PostScript 图像保存到文件。这对于创建表面的打印版本很有用。 示例： 
  postscript "surface.ps"

PRINT

主提示命令。打印表达式的值。 
  PRINT expr
打印表达式的值。这与 eprint 和 exprint 类似，但更简单。 示例： 
  print total_area
  print 2 + 3
  print "Hello, world!"

PRINTF

主提示命令。格式化打印。 
  PRINTF formatstring, args...
使用 C 语言风格的格式字符串打印输出。格式说明符包括：
  • %d - 整数
  • %f - 浮点数
  • %e - 科学计数法
  • %g - 紧凑浮点数
  • %s - 字符串
  • \n - 换行符
  • \t - 制表符
示例： 
  printf "Area: %f\n", total_area
  printf "Vertices: %d\n", vertex_count
  printf "Energy: %12.6e\n", total_energy
另请参阅 sprintf、errprintf。

QUADRATIC

主提示命令。将边设置为二次模式。 
  QUADRATIC
将边元素设置为二次模式。在二次模式下，每条边有一个中间顶点，允许更精确地表示表面。这对于 Hessian 能量最小化和优化很有用。 另请参阅 linear、lagrange。

QUIT

主提示命令。退出 Surface Evolver。 
  QUIT
退出 Surface Evolver 程序。这与键入 Ctrl-C 或在提示符下输入 q 的效果相同。

RAWV

主提示命令。原始顶点平均。 
  RAWV
执行原始顶点平均。这将每个顶点移动到其相邻顶点的质心。此命令不考虑约束或边界，因此主要用于调试和研究。 另请参阅 raw_vertex_average。

RAWESTV

主提示命令。最原始的顶点平均。 
  RAWESTV
执行最原始形式的顶点平均。这与 rawv 类似，但甚至不考虑元素权重。仅用于调试和研究。

RAW_VERTEX_AVERAGE

主提示命令。原始顶点平均。 
  RAW_VERTEX_AVERAGE
执行原始顶点平均。这与 rawv 相同，但使用更明确的名称。

RAWEST_VERTEX_AVERAGE

主提示命令。最原始的顶点平均。 
  RAWEST_VERTEX_AVERAGE
执行最原始形式的顶点平均。这与 rawestv 相同，但使用更明确的名称。

READ

主提示命令。从文件读取命令。 
  READ filenamestring
从文件读取命令并执行它们。文件中的命令就像在终端上输入它们一样。 示例： 
  read "commands.txt"

REBODY

主提示命令。重新分配面片到体。 
  REBODY
根据面片的几何位置重新分配面片到体。这对于修复拓扑错误很有用。

RECALC

主提示命令。重新计算所有量。 
  RECALC
重新计算所有几何量（面积、体积、能量等）。通常在修改表面后自动调用，但也可以手动调用以确保所有量都是最新的。

REFINE

主提示命令。细化元素集。 
  REFINE generator
细化由生成器指定的元素。对于边，这会在边的中点插入一个新顶点。对于面片，这会将三角形细分为四个三角形。细化是自适应网格生成的关键操作。 示例： 
  refine edges where length > 0.5
  refine facets where area > 0.1
  refine edges where on_constraint 1
另请参阅 equiangulate。

REPLACE_LOAD

主提示命令。替换当前表面。 
  REPLACE_LOAD filenamestring
用另一个数据文件替换当前表面，但不完全重新初始化。这与 addload 不同，后者会添加到当前表面。与 load 不同，replace_load 不会重置用户定义的变量和约束。

RENUMBER_ALL

主提示命令。重新编号所有元素。 
  RENUMBER_ALL
重新编号所有顶点、边、面片和体的 id。这会将所有 id 重新分配为从 1 开始的连续整数。这对于删除大量元素后清理 id 很有用。

REORDER_STORAGE

主提示命令。重新排列存储顺序。 
  REORDER_STORAGE
重新排列元素的存储顺序，以提高内存访问效率。这对于大型表面可以提高性能。

RESET_COUNTS

主提示命令。重置计数器。 
  RESET_COUNTS
重置所有内部计数器（如细化计数、边翻转计数等）。这与 flush_counts 类似，但不会保存到日志文件。

REVERSE_ORIENTATION

主提示命令。反转面片的方向。 
  REVERSE_ORIENTATION generator
反转由生成器指定的面片的方向。这会改变面片的法线方向。这对于修复拓扑错误很有用。

RITZ

主提示命令。使用 Ritz 方法计算特征值。 
  RITZ n
使用 Ritz 方法计算 Hessian 矩阵的前 n 个最小特征值和特征向量。此命令需要使用二次或 Lagrange 元素，并且需要线性约束。 输出包括找到的特征值列表以及每个特征值的特征向量范数。 示例： 
  ritz 5

SADDLE

主提示命令。执行鞍点搜索。 
  SADDLE
  SADDLE expr
执行鞍点搜索。如果提供了参数，则将其用作搜索方向的倍数。鞍点搜索尝试找到连接当前最小值与另一个最小值的鞍点。 这对于研究能量面和找到最小能量路径很有用。 另请参阅 hessian_seek。

SHOW_EXPR

T1_EDGESWAP

主提示命令。执行 T1 边交换。 
  T1_EDGESWAP generator
执行 T1 边交换操作。这与 edgeswap 类似，但适用于泡沫（foam）结构中的 T1 过程。在泡沫模拟中，T1 过程是当两个气泡交换邻居时发生的拓扑变化。 条件：

  VERTEX_AVERAGE
执行顶点平均。这将每个顶点移动到其相邻顶点的加权质心。权重基于相邻边和面片的几何属性。这有助于平滑表面。 另请参阅 rawv、rawestv。

VERTEX_MERGE

主提示命令。合并顶点。 
  VERTEX_MERGE expr
合并距离小于 expr 的顶点。这有助于清理几何体中的重复顶点。 示例： 
  vertex_merge 0.001

WHEREAMI

主提示命令。显示当前执行位置。 
  WHEREAMI
显示当前命令的执行位置（文件名和行号）。这对于调试脚本很有用。

WRAP_VERTEX

主提示命令。包裹顶点。 
  WRAP_VERTEX v_id,transform
将顶点 v_id 应用指定的对称变换。这对于在具有对称性的表面上移动顶点很有用。 示例： 
  wrap_vertex 1, 3

ZOOM

主提示命令。放大表面的一部分。 
  ZOOM facet_id
  ZOOM x,y,z,radius
放大表面的一部分。如果提供了一个面片 id，则放大该面片。如果提供了坐标和半径，则放大指定的球形区域。 示例： 
  zoom 5
  zoom 0.5, 0.5, 0.5, 0.3

重定向

命令输出可以通过 > 或 >> 重定向到文件。使用 > 会覆盖文件，使用 >> 会追加到文件。

  list vertices > "vertices.txt"
  printf "value: %f\n", total_area >> "area.txt"

输出也可以通过管道传输到系统命令，使用 |。

  list vertices | sort
  printf "%f\n", total_energy | grep "1.23"

命令输出还可以通过反引号捕获到字符串变量中。

  area_string := `printf "%f\n", total_area`
