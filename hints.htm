提示（HINTS）

这里汇集了我自身使用 Evolver 的经验以及帮助他人过程中积累的有用提示。
  • Evolver 使用无量纲单位（dimensionless units）进行运算，默认设置在尺寸、表面张力（surface tension）、体积（volume）等数值接近 1 时效果最佳。如果你选择的单位产生非常大或非常小的数值，可能需要调整诸如 scale_limit、 target_tolerance 和 constraint_tolerance 等参数。

  • 在绘制初始数据文件（datafile）的构造草图时，应尽可能画大一些，因为上面需要标注大量的标记。给所有顶点（vertex）、边（edge）和面片（facet）编号。在边上标注方向箭头，并标明面片的方向（我喜欢用围绕面片编号的弯曲箭头来表示）。

  • 初始面片应为凸形（convex）。虽然 Evolver 可以处理非凸面片，但其三角剖分（triangulation）算法非常简单，对非凸面片的三角剖分结果可能很差。只需添加一两条额外的边，将面片划分为几个凸面片即可。

  • 对于带有约束能量（constraint energy）或含量积分（content integrals）的边，以及不带这些的边，应分别创建独立的约束（constraints）。即使其他边是固定的，当只有精确需要的边附带有积分的约束时，检查被积函数（integrand）是否正确也会容易得多。

  • 如果你的所有元素没有连续编号（这通常是由于你使用的编号方案，或者添加或删除了元素），请使用 -i 命令行选项运行 Evolver，这样鼠标拾取（mouse-picking）报告的元素编号与数据文件中的一致。你也可以在数据文件顶部添加 "keep_originals" 来达到相同效果。

  • 确保你的所有体（body）的面片方向正确。如果普通边上的面片方向不匹配，Evolver 会报错，但固定边（fixed edges）、约束边（constrained edges）等不受此检查限制。一个很好的检查方法是使用着色，例如： 
    set body[1].facet color green

  • 确保顶点、边和面片位于其正确的约束上。你可以通过着色进行视觉检查，例如： 
    set edge color red where on_constraint 1
set facet color green where on_constraint 1
    你无法直接给顶点着色，但可以通过细化（refine）几次并给与顶点相邻的边着色来达到近似效果： 
    foreach vertex vv where on_constraint 1 do set vv.edge color blue

  • 检查你初始数据文件中的所有能量（energies）、体积（volumes）、量（quantities）等是否正确。有关如何进行详细检查的更多信息，请参见下面的章节。

  • 如果你要模拟带有固体壁上接触线（contact lines）的液体，我建议先创建一个数据文件，将液体的所有边界表面显式表示为面片，然后再创建第二个版本的数据文件，使用约束能量和含量积分来替代固定壁上的面片。在第一个版本中更容易得到正确的能量和体积，但它在演化过程中也更容易出问题。使用第一个版本来检查第二个版本的正确性，并使用第二个版本进行正式工作。

  • 如果你在弯曲约束（curved constraint）上的边试图抄近路（shortcut the curve），有几种方法可以阻止这种情况：
    • 创建第二条引导约束（guide constraint），使两条约束的交集为顶点定义运行轨道。通过使用顶点属性来自定义引导约束，你只需一条引导约束即可。例如： 
      define vertex attribute guides real[2]
constraint 1
formula:  x^2 + y^2 = rad^2   // curved constraint
constraint 2
formula:  guides[1]*x + guides[2]*y = 0  // radial guide planes
      然后你可以在运行时使用以下命令设置引导系数： 
      set vertex.guides[1] -y where on_constraint 1
set vertex.guides[1] x where on_constraint 1
    • 如果你确切了解是哪种能量或体积条件导致了抄近路行为，可以调整弯曲约束上的能量或含量被积函数来进行补偿，以消除这种倾向。这基本上意味着需要计算边与弯曲约束之间间隙的表面积，或间隙所围成的体积。
    • 将弯曲约束声明为 CONVEX（凸约束）。这会添加一个大致与间隙面积成比例的能量。这很简单，只要将 gap_constant 设置得足够高就能生效（但应将 gap constant 尽量保持在能工作的最低值），不过使用凸约束后你将无法使用任何 Hessian 命令。

  • 在细化之前先在低分辨率下运行。一个好的演化（evolution）脚本通常会交替进行细化和演化。三角形过多不仅计算耗时长，而且每次迭代（iteration）中运动只能传播一个三角形的距离。不要在某个特定的细化级别上过度演化。请记住这只是一种近似。将一个精度仅为 4 位的近似演化到 12 位精度是没有太大意义的。

  • 使用以下命令来整修（groom）你的表面三角剖分： V（顶点平均，vertex averaging）、 u（等角化，equiangulation）、 l（长边分割，long edge division）和 t（微小边删除，tiny edge deletion）。 可能需要一些实验才能找到正确的操作顺序，并与细化相结合。分割某些长边可能比简单地细化整个表面更好。然而，过度操作可能对收敛（convergence）产生反效果；有时收敛后的表面不希望被完全等角化或平均化，你可能会陷入迭代与整修的无限循环中。一旦你制定出一个好的脚本，请将其写入数据文件末尾的便捷命令中以便使用。

  • 使用 dump 或 d 命令定期保存你演化后的表面。请记住 Evolver 没有撤销（undo）功能来撤回灾难性的命令。

  • 使用共轭梯度（conjugate gradient）模式可以加快梯度下降（gradient descent）的速度，但不要过早使用。先使用常规梯度下降来适应体积或约束的变化。共轭梯度只应在常规运动稳定下来后才使用。共轭梯度假设能量函数是二次的，当情况不是这样时可能会出现问题。共轭梯度可能需要反复关闭和打开，以使其忘记之前的历史信息。

  • 在梯度下降（包括共轭梯度）过程中，请密切关注缩放因子（scale factor）。缩放因子应保持相对稳定。缩放因子趋向 0 并不意味着收敛；它意味着表面正在遇到困难。然而，合理的缩放因子可能取决于细化程度和其他因素。请参见合理缩放因子一节。

  • 二阶 Hessian 收敛（second-order Hessian convergence）比一阶梯度下降快得多，前提是 Hessian 方法有效。因此我的建议是，仅使用梯度下降来到达可以安全使用 hessian hessian_seek 的阶段。实际上，hessian_seek 几乎总是可以安全使用的，因为它会确保能量不断减小。我已经发现，在某些情况下，即使表面远未收敛，hessian_seek 作为迭代步骤也能取得惊人的效果。

  • 小心能量的鞍点（saddle point）。一个对称的表面，例如焊盘上或导线周围的一滴焊料，可能看起来已经通过梯度下降收敛了，但实际上只是到达了鞍点。使用 eigenprobe 命令来测试稳定性，如果不稳定，使用 saddle 命令来脱离鞍点。

  • 在梯度下降中判断收敛是很困难的。如果迭代以大致恒定的缩放因子运行，能量变化不大，并且在共轭梯度模式下长时间运行后也没有明显变化，那么你的情况可能已经很好。但请使用 eigenprobe 命令来确认， 并使用 hessian 来完成最终收敛。

  • 如果你打算使用二次模式（quadratic mode）或拉格朗日模式（Lagrange mode）来获得更高的精度，请先在线性模型（linear model）中演化至最终阶段，因为它速度更快，而且有更多可用的三角剖分整修命令。

检查你的数据文件

你应该始终检查你的初始数据文件，确保它完全按照你的意图运行。很容易出现被积函数符号错误，或者将量应用到错误的元素上。当你加载初始数据文件时，初始能量、体体积和量的值应该与你通过手工计算或你信任的另一个数据文件得到的预期值完全一致。特别是，当使用约束积分来替代省略的面片时，我建议你另外创建一个使用面片代替积分的独立数据文件，用于检查两者之间的一致性。

合理的缩放因子。

演化中的困难通常由较小的缩放因子发出信号，这意味着演化遇到了某种障碍。当然，这意味着你需要知道什么是合理的缩放因子，这取决于你使用的能量类型以及表面的细化程度。在正常演化中，缩放因子的大小由表面小尺度粗糙度的发展决定。结合一点量纲分析（dimensional analysis），可以得出结论：缩放因子应与 L2-q 成正比，其中 L 是典型的边长度，能量的单位是 lengthq。量纲分析如下：设 D 为一个顶点偏离平衡表面的扰动。一般而言，能量在平衡点附近是二次的，因此： 
E = D2Lq-2
因此该顶点处的能量梯度为： 
grad E = 2D Lq-2
运动是缩放因子乘以梯度，我们希望它与 D 成正比，所以： 
scale * grad E = scale * 2 D Lq-2 = D
因此缩放因子的量级为 L 2-q。以下是一些示例：
缩放因子的量纲依赖关系
能量类型能量量纲 缩放因子 示例文件
肥皂膜面积 L2 L0 quad.fe
弦长度 L1 L1 flower.fe
弦曲率的平方 L-1 L3 elastic8.fe
肥皂膜平均曲率的平方 L0 L2 sqcube.fe
特别是，面积演化的缩放因子与细化无关，但对于大多数其他能量，缩放因子会随细化而减小。

影响面积演化缩放因子的另一个常见因素是表面张力。在面片表面张力赋值为 470 的单位系统中进行液态焊料模拟，意味着所有计算得到的梯度都乘以 470，因此缩放因子需要减小 470 倍才能得到相同的几何运动。因此，你应该将 scale_limit 设置为表面张力的倒数。
