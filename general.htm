命令行选项

从系统命令提示符启动 Evolver 的语法如下： 

evolver [-ffilename] [-a-] [-d] [-e] [-i] [-m] [-pn] [-q] [-Q] [-w] [-x] [-y] [datafile]

程序将在当前目录和 EVOLVERPATH 中搜索数据文件。如果未找到数据文件，则会尝试添加 .fe 扩展名再次搜索。 在某些系统（Windows、Linux，可能还有其他系统）上支持通配符匹配，但使用通配符时需格外小心，因为可能出现意外的匹配结果。 如果仍未找到数据文件，或命令行中未指定数据文件，程序将提示用户输入。选项：

-a-

当出现需要转换的情况时，不启用自动转换到 命名方法和 物理量模式（用于调试）。

-d

在解析数据文件时打印 YACC 调试跟踪信息。如果您无法确定数据文件读取不正确的原因，并且您熟悉 YACC，这可能会有所帮助。

-e

回显输入。用于将管道输入的命令回显到屏幕，以便在 Evolver 由另一个进程控制时，用户可以跟踪程序的运行状态。

-ffilename

指定在读取数据文件后用作命令输入的文件名。在此文件结束后，输入将恢复为标准输入。其效果与从文件重定向输入相同，但 -f 会将命令回显到屏幕，并在文件结束后恢复为标准输入。另请注意，错误会导致输入恢复为标准输入。

-i

保持数据文件中列出的元素编号，而不是按顺序重新编号。可以通过在数据文件顶部放置关键字 keep_originals 来达到相同效果。

-m

在程序启动时开启内存调试。效果与 Evolver 的 memdebug 切换命令相同。

-pn

强制使用 n 个进程运行以多处理器模式编译的 Evolver。n 可以大于或小于物理可用的处理器数量。默认值为 1。此选项应视为实验性的；目前开销仍然过大，通常不太实用。

-q

在内部将所有内容转换为 命名物理量。目前仍有少数功能尚未实现相应的物理量；这些功能将产生错误信息。

-Q

禁止回显数据文件的 read 部分，以及使用 read 命令输入的文件内容；效果与 quietload 切换相同。

-w

使 Evolver 在出现任何警告时退出。适用于在 shell 脚本中运行 Evolver。

-x

使 Evolver 在出现任何错误时退出。适用于在 shell 脚本中运行 Evolver，以避免 Evolver 因等待错误响应输入而挂起。

-y

使 Evolver 在收到任何警告消息后停止执行命令并返回命令提示符。效果与 break_after_warning 运行时切换相同。

错误处理

当 Surface Evolver 检测到错误时，它会打印错误消息并尝试采取适当的措施。如果启动 Evolver 时使用了 -x 命令行选项，则 Evolver 会立即以非零错误代码退出。这在从 shell 脚本运行 Evolver 时非常有用。错误分为以下几类：

  • WARNING - 发生了您应该了解的情况，但 Evolver 在打印消息后会正常继续执行。通过设置 break_after_warning 切换，您可以使 Evolver 在当前命令或子命令结束后停止执行。通过设置 break_on_warning 切换，您可以使 Evolver 立即返回命令提示符。-y 命令行选项使 Evolver 在警告后退出，适用于批量运行。
  • SYNTAX ERROR - 解析器检测到错误。会打印出错误检测点之前的最近输入，但实际问题可能出现在更早的位置。
  • DATAFILE ERROR - 正在读取的数据文件中存在错误。Evolver 会尝试通过跳转到数据文件的下一个可识别部分来恢复，但在出现 5 次此类错误后将放弃该数据文件。已读取的表面数据可供检查，但它可能构成不一致的表面，不应尝试对其进行演化。
  • ERROR - 这是在命令执行过程中遇到的错误。命令被放弃，Evolver 返回主提示符。命令的操作不会被撤销。-x 命令行选项使 Evolver 在出现任何错误后立即退出；适用于批量运行。
  • FATAL ERROR - 无法恢复的错误。Evolver 立即退出。

表面初始化

每当 Surface Evolver 加载新表面时，无论是启动时还是响应 q 或 load 命令时，都会执行以下操作：

  • 任何先前的表面都会释放所有内存。所有用户定义的变量和命令都会被删除，包括当前正在执行的任何命令。
  • 内部变量初始化为默认值。
  • 读取数据文件。
  • 在 肥皂膜模型 中，任何非三角形面都会通过在中心创建一个顶点并从中心向各顶点创建边来进行三角剖分。
  • 在 肥皂膜模型 中，每条边周围的面片顺序通过几何方式确定。
  • 顶点被投影到任何 水平集约束 上。
  • 执行 C 命令中描述的检查。
  • 计算初始面积、能量和体积。
  • 视图矩阵 被重置，以使对象在图形窗口中居中。
  • 如果任何图形处于活动状态，则绘制新表面。
  • 启动主命令提示符。

中断执行

可以通过标准键盘中断（通常是 CTRL-C，Unix 专家们称之为 SIGINT）来中断 Evolver 的操作。在重复操作期间，这会设置一个标志，该标志在每个循环结束时进行检查。重复操作将在当前步骤后停止，控制将返回主提示符。如果在循环结束前再次中断，Evolver 将中止命令并返回主提示符。请注意，如果正在进行改变表面拓扑的操作，这可能会使表面处于不一致状态。如果在循环外部收到中断，也会立即中止。如果 Evolver 收到 SIGTERM（例如来自 Unix 的 kill 命令），它将转储到默认的 dump 文件并退出。这对于停止正在运行脚本的后台 Evolver 很有用。SIGHUP 也会发生同样的情况，因此丢失调制解调器连接将保存当前表面。

注意：在 Microsoft Windows 中，第二次中断不会产生太大效果，因为 Windows 会创建一个单独的线程来处理中断，而我找不到任何方法来强制停止有问题的线程并 longjmp 回它应该在的位置。因此，如果 Evolver 确实卡住了，您可能只需要终止整个程序。

并行计算

Surface Evolver 的并行版本可用于多种类型的系统，但这些版本是实验性的，实用性不强。

对于对称多处理器（同一台机器中有多个 CPU），-pn 命令行选项将使 Evolver 使用 n 个线程，因此最好使 n 不超过机器中的 CPU 数量。使用的线程数超过物理可用数量会降低速度，但可以在单处理器系统上运行多个线程进行测试。

目前，唯一并行执行的计算是 命名物理量，因此最好使用 -q 选项启动 Evolver。 Evolver 似乎受到内存带宽的限制，因此您可能无法获得预期的加速效果。

对于分布式计算，有一个 MPI 版本的 Evolver，但这仍然是实验性的，仅在非常特殊的情况下才有用。
