Surface Evolver 获取与安装

本章介绍如何获取和安装Evolver。 Evolver被设计为可在不同系统间移植。有Windows和Macintosh的预编译版本；对于unix/Linux系统，提供了源文件和Makefile。 各种系统的发行包可从 Evolver主页 获取。 每个包还包含文档、示例数据文件和脚本。文档子目录名为doc， 包含PDF格式的手册、文档的HTML版本（数学部分除外）以及简要的unix手册页evolver.1。 HTML文件也被Evolver的help命令使用。 示例位于子目录fe（这是我用于数据文件的文件扩展名；它代表"facet-edge"，指的是Evolver中表面的内部结构）。 以下是标准包的安装说明：
  • UNIX/Linux
  • Microsoft Windows
  • Macintosh OS X
PDF格式的完整手册可单独从 manual270.pdf 获取。

Unix版本

在unix风格系统上安装Surface Evolver的说明。 Evolver可在64位系统和32位系统上编译。

快速开始：
1. 下载 evolver-2.70.tar.gz。
2. 解压Evolver归档文件。
3. 在src子目录中，编辑Makefile，取消注释适用于您系统的行。
4. 运行"make"。
5. 通过运行"./evolver ../fe/cube.fe"进行测试。

详细说明：

1. Evolver以压缩tar归档文件 evolver-2.70.tar.gz 的形式分发， 可从http://www.susqu.edu/brakke/evolver获取。将此文件下载到工作目录。 打包的归档文件约2MB，解压后约5MB。 您可能还需要另外3或4MB的空间来编译。

2. 使用以下命令解压缩归档文件：

      gunzip evolver-2.70.tar.gz
提取文件：
      tar xvf evolver.tar
这将解压到三个子目录：src（源代码）、 doc（手册的html版本）和fe（示例数据文件和脚本）。 工作目录还将包含手册的PDF版本和手册页evolver.1。

3. 安装手册页：将evolver.1复制到manpath上的适当位置，例如/usr/local/share/man/man1。 您可能需要成为root才能获得执行此操作的权限。

4. 设置EVOLVERPATH环境变量： Evolver需要找到初始数据文件，有时还需要其他文件（例如 read 命令的命令文件和帮助文档文件）。如果指定的文件不在当前目录中，则会查询名为EVOLVERPATH的环境变量以获取目录搜索列表。数据文件目录和包含HTML文档文件的目录应 definitely 包含在内。格式与通常的PATH环境变量相同。像往常一样在您的系统中设置它，在.profile或.login或.cshrc或其他地方：

Unix C shell：

     setenv EVOLVERPATH /usr/you/evolver/fe:/usr/you/evolver/doc

Bourne shell： 

     EVOLVERPATH=/usr/you/evolver/fe:/usr/you/evolver/doc
     export EVOLVERPATH

5. 切换到Evolver目录的src子目录。

6. 为您的系统修改Makefile。Makefile以各种系统的宏定义集开始。如果您的系统已列出，请从适当行的开头删除注释符号'#'。如果您的系统不在那里，请使用GENERIC行，或设置您自己的。如果您自己定义，请确保在include.h中包含相应的部分。

7. 在Makefile中，编辑CFLAGS行以包含适当的选项（优化、浮点选项等）。

8. 在Makefile中，GRAPH应该是屏幕图形接口文件的名称。 如果可能，请使用glutgraph.o；现在大多数系统都有OpenGL/GLUT图形。 GLUT图形使用单独的线程来显示图形，因此如果您使用GLUT，必须在CFLAGS中放入-DPTHREADS，并在GRAPHLIB中放入-lpthread。 如果不使用GLUT，对于基本的X窗口图形，可以使用xgraph.o。 对于完全没有内置屏幕图形，可以使用nulgraph.o。 GRAPHLIB应该是适当的图形库加上任何其他需要的库。

9. 如果您想在多处理器机器上使用并行进程， 请在CFLAGS中放入-DPTHREADS，并在GRAPHLIB中放入-lpthread。目前，唯一并行完成的计算是基本能量和命名量。实际并行完成的进程数可以通过-p n命令行选项控制。

10. 如果您希望Evolver能够使用geomview， 请在CFLAGS中包含-DOOGL。

11. 如果您希望Evolver在高于默认最大值4的空间维度中操作，请在CFLAGS中包含-DMAXCOORD=n，其中n是最大空间维度。这设置了维度的上限，用于在数据结构中分配空间。

12. 如果您的系统支持long double数据类型，您可以通过在CFLAGS中编译-DLONGDOUBLE来以更高精度计算和打印值。但这会稍微减慢计算速度，应仅由精度狂热者使用。而且在Intel芯片上，它只进行80位精度，即使分配了128位存储，因为这是8087数学协处理器的硬件精度。对于128位精度（在软件中，因此慢得多），请使用-DFLOAT128编译并链接quadmath库（您必须自己找到）。

13. 如果您希望使用基于METIS分区软件的命令（metis、kmetis、body_metis和metis_factor），那么您应该从 http://www-users.cs.umn.edu/~karypis/metis/下载METIS包并"make"库libmetis.a（在某些系统上，make会抱怨找不到ranlib，但生成的libmetis.a仍然有效）。在Evolver的Makefile中，将-DMETIS添加到CFLAGS，并将-lmetis添加到GRAPHLIB。您可能还需要将-L metispath添加到GRAPHLIB以告诉链接器在哪里找到libmetis.a。请注意，METIS已包含在Windows可执行文件中。如果您在非常大的表面上使用hessian
