Surface Evolver 调试

Evolver 内置了多种调试功能。脚本调试面向普通用户；其他调试功能主要面向 Evolver 开发者（我）。
  • 脚本调试
  • 语法调试
  • 内存调试
  • 梯度调试
  • Hessian 调试
  • 迭代调试

脚本调试

 这里的“脚本”指的是任何 Evolver 命令，无论是在过程还是函数中。

调试可以通过传统方式完成，即在代码中插入打印语句。另一种方法是在脚本中使用 subcommand，让用户在脚本的特定点探索表面的状态。如果无法或不想修改脚本，则可以设置断点。

断点

用户可以在已加载的脚本中使用 "set breakpoint" 命令设置断点。语法为： 
BREAKPOINT scriptname linenumber
其中 scriptname 是函数或过程的名称，linenumber 是文件中要设置断点的行号。该行必须有可执行代码，否则会出错。linenumber 可以是表达式。

断点可以单独取消： 

UNSET BREAKPOINT scriptname linenumber
或批量取消： 
UNSET BREAKPOINTS

当遇到断点时，Evolver 将进入子命令提示符，用户可以在其中输入任何 Evolver 命令（尽管某些命令如 load 非常不明智）。要退出子命令提示符，使用 q 或 exit 或 quit。

单步执行：在调试提示符下，有一个特殊的单步命令 n，它将继续执行到下一行（准确地说，直到行号改变）。

堆栈跟踪：在调试提示符下，whereami 命令将打印堆栈跟踪，显示到达当前位置的函数或过程调用序列。

语法调试

 Evolver 命令解析器生成的错误消息试图提供帮助，但有时相当晦涩。常见错误包括关键字拼写错误、变量拼写与关键字相同、使用未初始化的变量以及缺少分号。在尝试这里描述的花式语法调试之前，请先检查这些。可以打印出解析步骤，但这不适合胆小者，因为输出采用非常不友好的 YACC 格式。您必须了解移位-归约解析的工作原理才能理解输出。debug 开关控制 YACC 跟踪功能。示例：

Hessian 调试

 hessian_quiet 开关可以关闭，以打印 Hessian 分解期间（hessian、hessian_seek、ritz、eigenprobe 命令）的内存使用和工作量信息。例如： 
Enter command: hessian_quiet off
Enter command: eigenprobe 0
Sparse init alloc: 36897
Expanded hashtable size: 73794.
Expanded hashtable size: 147588.
Sparse entries: 61444  Final hashtable size: 147588
Hash extra probes: 256957
Variables: 12291  Original fill: 61446
Workspace: 8 bytes
Passes through main loop: 34
Total_fill:  538907
Total_flops: 7.25226e+007
Eigencounts:    3 <,  0 ==,  12286 >
根据 ysmp 开关是否开启（使用 Yale 稀疏矩阵包）或关闭（使用我自制的最小度算法），输出会有一些差异。

仅限专家。 Evolver 有一个功能，允许对命名量的 Hessian 计算例程的正确性进行数值检查。hessian_diff 开关使 Hessian 通过梯度的有限差分而不是相关例程来计算。数值 Hessian 在某些类型的表面上有一些限制，主要是不能用于体积或量约束。但由于任何量都可以作为能量测试，这并不影响其主要目的。

用于调试和调整我的最小度算法的一些内部变量：

mindeg_debug_level

控制调试消息的详细程度。

mindeg_min_region_size

期望的最小区域大小；较小的区域将与父节点合并。默认值 0。

mindeg_margin

在寻找好的消去时，高于最小度的范围。默认值 5。
