杂项术语

本文件提供了一些关键词的条目，这些关键词由于歧义、极其晦涩或其他原因，在其他地方不作为名称指针使用。

Alice

一个未记录文档的特殊用途命令的关键字。

Area（面积）

作为关键字，area 是面（facet）的只读属性。

Area_fixed（固定面积）

在数据文件中声明固定表面总面积的过时方法。同义词：fixed_area。现在推荐的方式是在数据文件中定义一个固定的 命名量，例如：
quantity the_area fixed = 2.3 global_method facet_area
这样可以使用所有命名量的功能。

Attribute（属性）

作为关键字，"attribute" 用于 define 命令中定义元素的 额外属性。

Bare（裸露）

作为关键字，"bare" 是一种 顶点属性或 边属性，由用户设置，用于告知 Evolver 不期望该顶点或边与面相连。

Body（体）

体是一种三维几何元素。 作为命令关键字，"body" 是 元素生成器中的元素名称，既可以作为主元素类型，也可以作为面的子元素。

Burchard

一个未记录文档的特殊用途命令的关键字。

Color（颜色）

作为关键字，"color" 是边和面的属性。 有关允许的值，请参见 颜色。

Conducting_knot_energy（导电结能量）

数据文件关键字，自动创建一个使用 knot_energy 方法作为全局方法的命名能量量。 在数据文件中可以后跟一个模量值。

Insulating_knot_energy（绝缘结能量）

数据文件关键字，自动创建一个使用 uniform_knot_energy 方法作为全局方法的命名能量量。 在数据文件中可以后跟一个模量值。

Constraint（约束）

"Constraint" 可以在 一般意义上使用，但作为关键字，它指的是 水平集约束。 "Constraint" 和 "constraints" 可以互换使用。

Content（含量）

"Content" 用于表示体积（或在弦模型中为面积），见于 约束积分中。

Convex（凸）

在数据文件中声明 约束时使用的关键字，会包含 间隙能量。 其目的是防止直线边和面以大间隙绕过弯曲约束。

Density（密度）

关键字，指体的重力密度、面的表面张力或边的线性张力。 在后两种情况下，"density" 与 "tension" 同义。

Edge（边）

一般来说，是基本几何元素之一。作为关键字，它用于 元素生成器中。 同义词：edges。

Efixed

过时的数据文件属性，用于在不固定端点的情况下固定边。 保留该关键字仅为与旧转储文件兼容。

Eprint

打印表达式并返回值的函数。语法：eprint expr。 用于调试；可能是命令语言尚未完善时的遗留产物。 示例：print sum(facet, eprint area) 将打印所有面的面积，然后打印总和。

Facet（面）

一般来说，是基本几何元素之一。在肥皂膜模型中，它是由三条有向边定义的三角形。在弦模型中，它是任意数量边的连接序列。在单纯形模型中，它是由数据文件顶部的 surface_dimension 项指定维度的单纯形，并由其顶点定义。

Fixed（固定）

可应用于 顶点、 边、 面或 命名量的属性。

Formula（公式）

数据文件关键字，用于 水平集约束中引入函数公式。同义词：function

Gauss_curvature（高斯曲率）

数据文件关键字，自动创建一个使用 gauss_curvature_integral 方法作为全局方法的命名能量量。 在数据文件中可以后跟一个模量值。

Tolerance（容差）

作为关键字，"tolerance" 指的是 固定量属性， 用作收敛的判据。默认值使用 target_tolerance。

Div_normal_curvature（散度法向曲率）

切换开关，使 sq_mean_curvature 能量通过面顶点处法向量的散度来计算平均曲率。

Force_pos_def（强制正定）

切换开关。如果在 Hessian 矩阵的 YSMP 分解期间开启此项，且 Hessian 矩阵变为不定，则会在对角元素上添加某些值以使其正定。可能是某次实验的遗留。

H_inverse_metric（H 逆度量）

切换开关。用力的拉普拉斯算子替代力。用于进行平均曲率的拉普拉斯运动。

Modulus（模量）

关键字，是 命名量或 方法实例的属性，作为计算值的乘数。

Opacity（不透明度）

在某些 geomview 系统上用于透明度的面属性。 语法：set facet opacity value，其中 value 在 0 和 1 之间。实际上，这只是为所有面同时设置一个全局变量。该值在颜色的 alpha 通道中传递给 geomview。

Parameter（参数）

指 参数化边界参数，或用户定义的 变量。

Parameter_file（参数文件）

在数据文件的顶部区域，可以用文件中的一组值初始化变量，语法为 
PARAMETER name PARAMETER_FILE string
我忘记了具体的工作方式。

Pressure（压力）

通常指 固定体积体或 固定命名量的属性，实际上是约束的拉格朗日乘子。 作为数据文件中的体属性，为体建立 固定压力。 也偶尔用于数据文件的顶部区域以建立 理想气体模型。

Surface_energy（表面能量）

在数据文件中包含矢量表面能量积分的过时方法。当前方法是使用带有 facet_vector_integral 方法的 命名量。

Total（总计）

命名量值（例如 my_quant）的过时语法是 "total my_quant"。当前语法是 "my_quant.value"。

Valence（价）

顶点的属性，即关联边的数量；或边的属性，即关联面的数量；或面的属性，即面上边的数量。

Vertex（顶点）

基本 几何元素之一。 作为关键字，用于 元素生成器中。

Utest

运行测试以查看三角剖分是否为 Delaunay 三角剖分。用于高维和单纯形模型。

Stability_test（稳定性测试）

用于查找 迁移率矩阵最大特征值的命令。不太记得其用途。

Tag（标签）

过时的面属性，曾经用于将任意值与面关联。现已被 额外属性取代。

动态加载库

此功能使用不多，但为完整起见在此提供文档。 许多 Evolver 功能，如 水平集约束、 参数化边界、 命名方法被积函数和 黎曼 度量，都需要一组参数的用户定义函数。这些函数的表达式通常存储为解析树，并在每次需要时解释执行，这比求值编译后的表达式要慢得多。有一种方法可以通过称为动态加载的机制使用特定于数据文件的编译函数集。这里，数据文件的函数库被单独编译，然后在运行时加载数据文件时加载。目前，Evolver 只实现了许多 Unix 系统上存在的动态加载机制，可以通过检查文件 /usr/include/dlfcn.h 是否存在来测试。如果存在，可以通过在 Makefile 的 CFLAGS 行中包含 -DENABLE_DLL 来启用动态加载。在某些系统上，可能还需要在 GRAPHLIB 行中包含 -ldl，以便将 Evolver 与 dlopen() 等函数链接。 要为数据文件创建库，请编写一个包含所需函数的 C 源代码源文件，编译并将其链接为共享库。该函数应能够计算函数的值和偏导数，如果要使用任何 Hessian 功能，还应计算二阶偏导数。一个二维数据文件的示例源文件： 

#define FUNC_VALUE  1
#define FUNC_DERIV  2
#define FUNC_SECOND 3
#define MAXCOORD 4   /* must be same as in Evolver!! */
#define REAL double  /* long double if Evolver compiled with -DLONGDOUBLE */
struct dstack { REAL value;
		REAL deriv[2*MAXCOORD];
                REAL second[2*MAXCOORD][2*MAXCOORD]; };

void func1 ( mode, x, s )
int mode; /* FUNC_VALUE, FUNC_DERIV, FUNC_SECOND */
REAL *x;  /* pointer to list of arguments */
struct dstack *s;  /* for return values */
{ REAL value;

  s->value = x[0] + x[1]*x[1];

  if ( mode == FUNC_VALUE ) return;

  /* first partials */
  s->deriv[0] = 1.0;
  s->deriv[1] = 2*x[1];

  if ( mode == FUNC_DERIV ) return;

  /* second partials */
  s->second[0][0] = 0.0;
  s->second[0][1] = 0.0;
  s->second[1][0] = 0.0;
  s->second[1][1] = 2.0;

  return;
}
假设源文件名为 foo.c， 在 SGI 系统上（IRIX 5.0.1 或更高版本）编译和链接： 
cc -c foo.c
ld -shared foo.o -o foo.so
Sun 系统相同，但用 -s 替代 -shared。对于其他系统，请查阅 ld 文档以获取创建共享库或动态加载库的选项。

要在数据文件中使用这些函数，在数据文件顶部在使用任何函数之前包含一行： 

load_library "foo.so"
将在当前目录和 EVOLVERPATH 中搜索该库。 最多可以加载 10 个库。 之后，可以通过仅使用函数名来调用任何函数，无需显式参数列表，因为在这些函数合法使用的地方参数列表始终是隐式的。示例，假设 func2 也定义了一个参数： 
constraint 1
formula: func1

boundary 1 parameters 2
x1: func2
x2: 3*func2 + sin(p1)
您需要确保函数期望的参数数量与函数使用中隐含的数量相同。 无需在数据文件中显式声明函数。任何未定义的标识符都会被检查是否为动态加载的函数。

注意：此动态加载实现是实验性的，此处描述的接口将来可能会更改。

__bhead_index

__vhead_index

Hessian 例程内部使用的顶点属性。

form_factors（形态因子）

多种弹性能量命名方法使用的面额外属性向量： linear_elastic、 neo_hookean、 relaxed_elastic、 relaxed_elastic_A、 SVK_elastic 和 dirichlet_elastic。 各分量是松弛面各边的点积。由于对称性，只有三个不同的点积，因此 form_factors 的长度仅为 3。各分量为 {s11,s12,s22}，其中 s11 = dot(v2-v1,v2-v1)，s12 = dot(v2-v1,v3-v1)，s22 = dot(v3-v1,v3-v1)。如果用户未定义 form_factor，Evolver 将创建它，并假设初始面形状为无应变状态。

poisson_ratio（泊松比）

各种弹性命名方法使用的面额外属性： linear_elastic、 linear_elastic_B、 neo_hookean、 relaxed_elastic、 relaxed_elastic_A、 SVK_elastic 和 dirichlet_elastic。 这是二维各向同性泊松比。
某些物理表面在平均曲率为某个非零值时处于最低能量状态。一个例子是由平行堆叠分子制成的膜，这些分子的一端比另一端大。因此，其曲率能量与 (h-h0)^2 成正比，其中 h 是实际平均曲率，h0 是平衡或固有平均曲率。几种平方平均曲率命名方法可以使用 h0，形式为变量 h_zero 或实值顶点属性 h_zero。如果顶点属性存在，则优先使用。 使用 h_zero 的方法：
  • sq_mean_curv_cyl
  • sq_mean_curvature
  • eff_area_sq_mean_curvature
  • normal_sq_mean_curvature
  • star_normal_sq_mean_curvature
  • star_perp_sq_mean_curvature
  • sqcurve2_string
不使用 h_zero 的方法：
  • star_sq_mean_curvature
  • star_eff_area_sq_mean_curvature
  • sqcurve_string
  • sqcurve_string_marked
  • sqcurve3_string
  • circle_willmore
