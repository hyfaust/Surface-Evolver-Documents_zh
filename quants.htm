命名量和方法

这是计算全局量（如面积、体积和表面积分）的系统化方案， 替代了 Evolver 中原始的临时方案。简而言之， 方法是内置函数， 而命名量是 方法实例的组合。 参见 ringblob 数据文件获取示例。 原始的临时计算在存在的地方仍然是默认的，但所有新的量都以命名量方案添加。 某些新功能仅适用于命名量。要将所有内容转换为命名量， 请使用 -q 选项启动 Evolver， 或使用 convert_to_quantities 命令。 这尚未设为默认值，因为命名量可能比原始方法慢。

示例数据文件 qcube.fe、qmound.fe 和 ringblob.fe 包含一些命名量和实例的示例。前两个是 cube.fe 和 mound.fe 的量版本。 这些说明了最通用和有用的方法，即 facet_vector_integral、facet_scalar_integral 和 edge_vector_integral， 而不是更快但更专业的方法，如 facet_area。 我的建议是，用户将旧的隐式方法用于面积、体积和重力能量， 仅在特殊情况下使用命名量。

命名方法

"方法"是一种从特定类型的元素（顶点、边、面、体）计算标量值的方式。 每种方法在内部实现为一组函数，用于根据顶点位置计算值及其梯度。 最常见的方法也有海森矩阵函数。 方法通过其名称引用。

参见 已实现的方法获取可用方法列表。 添加新方法涉及编写 C 程序来计算值和梯度（可能还有海森矩阵） 作为顶点坐标的函数，将函数声明添加到 quantity.h， 并将结构添加到 quantity.c 中的方法声明数组。 从数据文件调用它的所有其他语法已经就位。

方法实例

"方法实例"是将特定方法应用于特定几何元素集的总和。 某些方法（如 facet_area）是完全自包含的。其他方法（如 facet_vector_integral） 需要用户指定一些进一步的信息。对于这些方法，每个实例都有此进一步信息的规范。 方法实例在数据文件中定义，可以是 命名量定义的未命名部分， 也可以是包含在命名量中的单独 命名方法实例。 单独的命名版本在您想要检查整个表面或单个元素的实例值时非常有用。 实例总值可以使用 A 命令打印， 或作为 "instancename.value" 在命令中引用。实例名称本身可以用作元素属性。 例如，假设有一个名为 moment 的实例，它应用于面。 那么典型的命令将是 
  print moment.value
   print facet[3].moment
   list facet where moment > 0.1
模数。 每个方法 实例都有一个"模数"，它乘以基本方法值以给出实例值。模数为 0 会在计算量时导致整个实例计算被省略。模数可以在 数据文件中设置， 或使用 A 命令或通过赋值设置。示例命令： 
   print moment.modulus
   moment.modulus := 1.3
方法实例可以声明为对每个元素使用不同的模数， 通过指定用于该目的的额外元素属性。额外属性必须已经声明。示例： 
   define facet attribute mymod real
   quantity myquant energy method facet_area global element_modulus mymod
当然，由用户正确初始化额外属性的值。

方向。 某些方法（逻辑上依赖于元素方向的方法）可以应用相对方向。 当在数据文件中应用于单个元素时，负方向通过在实例名称后添加 '-' 来表示。 当在运行时使用 set 命令应用时， 如果元素以负方向生成，则方向将为负， 即 set body[1].facet method_instance qqq。 当前实现此功能的方法有： edge_vector_integral、string_gravity、facet_vector_integral、 facet_2form_integral、facet_volume、facet_torus_volume、 simplex_vector_integral、simplex_k_vector_integral、 edge_k_vector_integral、gravity_method 和 full_gravity_method。

命名量

"命名量"是各种 方法实例的总和， 尽管通常只涉及一个实例。 实例不需要应用于相同类型的元素； 例如，定义体积量可能需要面积分和边积分。 每个命名量都是四种类型之一：
  • "energy" 量，它们被添加到表面的总能量中；
  • "fixed" 量，它们被约束到固定目标值（通过每次迭代的牛顿步）；以及
  • "conserved" 量类似于 fixed，但值无关紧要。量梯度用于消除运动中的一个自由度。 很少使用，但可用于消除旋转自由度，例如。 由于它们通过差分进行梯度计算，因此不能与优化参数一起使用。
  • "info_only" 量，其值仅向用户报告。
此类型在量的 数据文件声明中初始设置。量可以使用 "fix quantityname" 和 "unfix quantityname" 命令在 fixed 和 info_only 之间切换。

量的值可以使用 A 或 v 命令显示， 或作为表达式 "quantityname.value"。此外，使用 量名称作为元素属性会评估为该元素上所有适用组件实例值的总和。例如， 假设有一个名为 vol 的量，可以执行 

 
   print vol.value
   print facet[2].vol
   histogram(facet,vol)

模数。 每个量都有一个"模数"，它只是所有实例值总和的标量乘数。模数为 0 将关闭所有实例的计算。模数可以在 数据文件声明中设置， 使用 A 命令，或通过赋值设置： 

 quantityname.modulus := 1.2

目标值。 每个固定量都有一个目标值，Evolver 尝试将量值约束到该目标值。 每次进行迭代时（g 命令或各种 海森 命令）， 使用牛顿法将表面投影到约束值。目标值可以使用 A 或 v 命令显示，或作为 "quantityname.target"。 可以使用 A 命令或通过赋值更改。示例： 

   print qname.target
   qname.target := 3.12

体积常数。 量可以添加一个常数值，类似于 体属性 volconst。 此量属性也称为 volconst。它可用于添加已知积分值， 例如从实际计算中省略的积分。它可以在量的 数据文件 定义中设置， 或通过 赋值 命令设置。

压力。 每个固定量都有一个与之关联的拉格朗日乘数。 约束的拉格朗日乘数是能量相对于约束目标值的变化率。 对于体积约束，拉格朗日乘数就是压力。 拉格朗日乘数在执行迭代步骤时计算。它们可以在 "pressure" 列中使用 v 命令显示，或作为表达式 "quantityname.pressure"。

容差。 固定量可以具有容差属性，用于判断收敛性。 当所有量比值与容差的总和小于 1 时，表面被认为已收敛。 此总和还包括固定体积的体。如果未设置容差或为负值， 则使用变量 target_tolerance 的值，其默认值为 0.0001。

函数量。 命名量可以是命名方法值的任意函数，而不是简单的方法总和。 数据文件语法使用 "function expression" 代替方法列表。 例如： 

   method_instance qwerty method facet_scalar_integral
   scalar_integrand: x^2
   quantity foobar energy function qwerty.value^3
注意方法名称使用后缀 "value"。还要注意， 使用的方法是全局值，而不是逐元素值。 如果组件方法有海森矩阵，函数量可以执行海森操作。 此类海森矩阵在默认稠密矩阵形式下可能变得非常消耗内存； 有一个切换命令 function_quantity_sparse 将导致使用稀疏矩阵。

方法描述

以下对各个方法的描述给出了方法的数学定义、 它应用于的元素类型、定义参数、 它适用于的模型类型、 对环境空间维度的任何限制，以及 方法是否实现了海森矩阵。除非特别说明， 方法已实现梯度，因此可用于能量或约束。定义参数通常是标量或向量被积函数（完整语法参见 数据文件声明）。 某些方法还依赖于全局变量，如注释所示。 给出的示例数据文件声明适用于简单情况； 完整语法在其他地方给出。 请记住，在示例中，对于未声明为全局的量， 量必须单独应用于所需的元素。

0维

vertex_scalar_integral

命名方法。 描述：顶点处的函数值。这实际上产生顶点上的总和， 但作为数学家，我将顶点上的总和视为点加权积分。
元素：顶点。
参数：scalar_integrand。
模型：线性、二次、拉格朗日、单纯形。
环境维度：任意。
海森矩阵：是。
示例数据文件声明： 
   quantity point_value energy method vertex_scalar_integral
   scalar_integrand: x^2 + y^2 - 2x + 3

模型：线性、二次、拉格朗日。
环境维度：任意。
海森矩阵：是。
示例数据文件声明：

edge_general_integral

命名方法。 描述：位置和切向的标量函数在边上的积分。 切向向量的分量通过继续坐标索引来表示。也就是说，在 3D 中， 位置坐标是 x1,x2,x3，切向分量是 x4,x5,x6。 为了获得正确的行为，被积函数应该在切向分量中是 1 次齐次的。 使用阶数为 integral_order_1D 的高斯求积。
元素：边。
参数：scalar_integrand。
模型：线性、二次、拉格朗日。
环境维度：任意。
海森矩阵：是。
示例数据文件声明： 3D 中的边长可以用此量计算： 
   quantity arclength energy method edge_general_integral
   scalar_integrand: sqrt(x4^2 + x5^2 + x6^2)

海森矩阵：是。
示例数据文件声明：

海森矩阵：是。

hooke_energy

命名方法。 描述： 人们通常希望要求边具有固定长度。 可以通过定义固定量来约束某些边的总长度。 这用于固定演化纽结的总长度，例如。 但是为每条边设置一个约束是不切实际的， 因为投影到 n 个约束需要求逆 n x n 矩阵。 相反，可以使用胡克定律能量来鼓励边具有相等的长度。每条边的形式为 
   E =  | L - L_0| ^p
其中 L 是边长， L_0 是平衡长度，体现为可调参数 `hooke_length'， 幂 p 是可调参数 `hooke_power'。默认幂为 p = 2， 默认平衡长度是初始数据文件中的平均边长。 您需要调整此值，特别是如果您有总长度约束。 高模数会减少总能量的胡克分量， 因为恢复力与位移成线性关系，而能量是二次的（当 p=2 时）。 作为额外奖励， `hooke_power' 为 0 将给出 
   E = -\log|L-L_0|
参见 hooke2_energy 获取单个边平衡长度。
元素：边。
参数：无。
模型：线性。
环境维度：任意。
海森矩阵：是。
示例数据文件声明： 
   parameter hooke_length 0.3   // 将应用于所有边
   parameter hooke_power  2     // 默认值
   quantity slinky energy method hooke_energy global

hooke2_energy

命名方法。 描述：与 hooke_energy 相同， 但每条边都有一个平衡长度 额外属性 `hooke_size' （用户无需声明）。 如果用户在方法首次调用之前未设置 hooke_size， 则该值将默认为当前长度。 Hooke_size 不会通过细化自动调整。 用户有责任在细化后重置 hooke_size；您可以重新定义 'r' 命令 
   r :::= { 'r'; set vertex hooke_size hooke_size/2 }
来自动处理它。 使用的位移幂由内部读写变量 hooke2_power 给出，其默认值为 2。
元素：边。
参数：无。
模型：线性。
环境维度：任意。
海森矩阵：是。
示例数据文件声明： 
   parameter hooke2_power  2     // 默认值
   define edge attribute hooke_size real
   quantity slinky energy method hooke2_energy global
   ...
   read
   r;r;set edge hooke_size length

hooke3_energy

命名方法。 描述：与 hooke2_energy 相同， 但使用弹性模型而不是弹簧。能量为 
energy = 0.5*(length-hooke_size)^2/hooke_size。
指数可以通过设置参数 hooke3_power 从 2 改变。 如果内部变量 frickenhaus_flag 非零，则如果长度小于平衡长度，能量被视为 0。
元素：边。
参数：无。
模型：线性。
环境维度：任意。
海森矩阵：是。
示例数据文件声明： 
   parameter hooke3_power  2     // 默认值
   quantity slinky energy method hooke3_energy global
   ...
   read
   r;r;set edge hooke_size length

local_hooke_energy

命名方法。 描述： 边作为弹簧的能量， 平衡长度是相邻边长度的平均值。 实际上，能量是按顶点计算的， 
 E = ({L_1 - L_2 \over L_1 + L_2})^2
其中 L_1 和 L_2 是顶点相邻边的长度。用于字符串循环。（John Sullivan 提供） 如果设置变量 local_hooke_flag 非零，则 local_hooke_energy 不会在价数不等于 2 的顶点处评估。
元素：顶点。
参数：无。
模型：线性。
环境维度：任意。
海森矩阵：否。
示例数据文件声明： 
   quantity slinky energy method local_hooke_energy global

dihedral_hooke

元素：边。
参数：无。
模型：线性。
环境维度：任意。
海森矩阵：是。

命名方法。 曲线的平方扭率积分。 扭率通过查看相邻三条边的三元组来近似；如果 A,B,C 是边向量，则密切平面扭转角度的正弦（从 AxB 到 BxC）为 
       [A,B,C] |B|
  S =  -----------
       |AxB| |BxC|

sq_mean_curvature

命名方法。 描述：表面平方平均曲率的积分。 有几种方法用于计算表面平方平均曲率的积分。 旧方法 sq_mean_curvature、 eff_area_sq_mean_curvature 和 normal_sq_mean_curvature 现在已弃用， 因为它们没有海森矩阵，而较新的方法 star_sq_mean_curvature、 star_eff_area_sq_mean_curvature、 star_normal_sq_mean_curvature 和 我目前最喜欢的 star_perp_sq_mean_curvature 有 海森矩阵，现在可以处理顶点周围不完整的面星。但请阅读以下关于平方曲率的一般说明。

肥皂膜模型中平方平均曲率的积分计算如下： 每个顶点 v 周围有一个面积为 A 的面星。 由于表面张力作用在顶点上的力 F 是面积的梯度， 由于每个面有 3 个顶点，与 v 相关的面积是 A/3。因此顶点处的平均曲率是

h = (1/2)(F/(A/3)),
其中 1/2 因子来自"平均"部分的"平均曲率"。 然后该顶点对总积分的贡献是
E = h2A/3 = (3/4)F2/A。
哲学说明：三角化表面上的平方平均曲率在技术上是无限的， 因此需要某种近似方案。 将曲率定位在顶点的替代方案是将其定位在边上， 它实际上在那里，并将其平均到相邻的面。但这有一个问题， 即最小面积三角化表面将具有非零平方曲率， 而在顶点公式中它将具有零平方曲率。

实际说明：上述平方平均曲率的定义在实践中似乎容易出现不稳定。 一个是尖角变得更尖锐而不是平滑。另一个是某些面想要以牺牲邻居为代价变得非常大。 因此添加了几种替代定义。
边界处的曲率：如果表面边缘是约束上的自由边界， 则上述计算给出正确的曲率，假设表面通过反射穿过约束继续。 这允许通过对称区域表示对称表面。如果表面边缘是固定边或在一维边界上， 则无法从相邻面的知识计算边界顶点处的曲率。 例如，悬链面和球冠底面周围的面环可能是相同的。 因此曲率仅在内部顶点处计算，当进行表面积分时， 边界上的面积被分配给最近的内部顶点。 但是，在方法声明中包含 IGNORE_FIXED 或 IGNORE_CONSTRAINTS 将分别在固定点或忽略约束的情况下强制计算能量。
如果定义了参数或顶点属性 h_zero， 则每个顶点的值与以下方法 eff_area_sq_mean_curvature 相同。

元素：顶点。
参数：IGNORE_CONSTRAINTS、IGNORE_FIXED。
模型：线性。
环境维度：任意。
海森矩阵：否。
示例数据文件声明： 

   quantity sqc energy method sq_mean_curvature global

eff_area_sq_mean_curvature

命名方法。 描述：表面平方平均曲率的积分，定义与 sq_mean_curvature 或 normal_sq_mean_curvature 略有不同。 顶点周围的面积取为体积梯度的大小。这小于真实面积，因此产生更大的曲率。 这也消除了尖峰不稳定性，因为尖峰具有更大的面积梯度但相同的体积梯度。设 N 为顶点 v 处的体积梯度，
h = (1/2)(F/N)),
和
E = h2 A/3 = (3/4)(F·F/N·N)A.
为了使此方法有效，面的方向必须一致， 因为 evolver 需要表面的"内部"和"外部"来计算体积梯度。仍然可能存在某些面以牺牲其他面为代价增长的不稳定性。
如果定义了参数或顶点属性 h_zero，则每个顶点的值是
E = (h-h0)2 A/3 = (3/4)(F·N/N·N-2h0)2A.
当 h_zero 具有零值时，这不会简化为非 h_zero 公式， 但它实际上本身就是一个相当好的公式（参见 star_perp_sq_mean_curvature 。
如果顶点在一个或多个约束上，F 和 N 将投影到约束上， 本质上使约束充当镜像对称平面。如果不应将约束视为镜像平面， 则应在数据文件的声明中为其赋予属性 nonwall。
警告：对于某些极端形状， Evolver 可能难以检测一致的局部表面方向。 assume_oriented 切换允许 Evolver 假设面已定义为一致的局部方向。

元素：顶点。
参数：无。
模型：线性。
环境维度：任意。
海森矩阵：否。
示例数据文件声明： 
   quantity effsq energy method eff_area_sq_mean_curvature global

normal_sq_mean_curvature

命名方法。 描述：表面平方平均曲率的积分，定义与 sq_mean_curvature 或 eff_area_sq_mean_curvature 略有不同。 为了缓解 eff_area_sq_mean_curvature 的不稳定性， normal_sq_mean_curvature 将顶点周围的面积视为体积梯度平行于平均曲率向量的分量， 而不是体积梯度的大小。 因此

linear_elastic

命名方法。 描述： 基于 Cauchy-Green 应变矩阵计算面的各向同性线性弹性应变能量。 设 S 为未应变面的 Gram 矩阵（边的点积）。 设 Q 为 S 的逆矩阵。 设 F 为应变面的 Gram 矩阵。 设 C = (FQ-I)/2，Cauchy-Green 应变张量。 设 v 为泊松比。 则能量密度为
(1/2/(1+v))(Tr(C^2) + v*(Tr C)^2/(1-(dim-1)*v))
每个面都有额外属性 poisson_ratio 和 额外属性数组 form_factors[3] = {s11,s12,s22}， 它们是 S 中的条目。也就是说，s11 = dot(v2-v1,v2-v1)， s12 = dot(v2-v1,v3-v1)，s22 = dot(v3-v1,v3-v1)。 如果用户未定义 form_factor，它将由 Evolver 创建， 初始面形状将被假设为未应变。 对于此方法的版本，使压缩零能量，参见 relaxed_elastic_A。
元素：面。
参数：无。
模型：线性。
环境维度：3。
海森矩阵：是。
示例数据文件声明： 
   quantity lastic energy method linear_elastic global

general_linear_elastic

命名方法。 描述： 计算面的非各向同性线性弹性应变能量。 设 A 为从未应变形状到应变形状的线性变换。则 Cauchy-Green 应变张量为 C = (ATA - I)/2。设 S1 和 S2 为未应变面的边。 设 W1 和 W2 为变换后的面边。设 F 为应变面的 Gram 矩阵。 定义
S = [ S1 S2 ], Q = S-1
W = [ W1 W2 ] = AS
F = WTW = STATAS
则
ATA = QTFQ
C = (QTFQ - I)/2
能量密度为
(1/2)Cij Kijkl Ckl
其中 Kijkl 是弹性常数的完整张量。 通过使用对称性，这可以简化为
(1/2) [ C11 C22 C12 ] [ E1 E3 E4 ] [ C11 ]
[ E3 E2 E5 ] [ C22 ]
[ E4 E5 E6 ] [ C12 ]
每个面都有大小为 6 的额外属性 elastic_coeff 包含 { E1, E2, E3, E4, E5, E6 }，和 大小为 2x2 的额外属性数组 elastic_basis 包含 { {s11,s12},{s21,s22}}， 它们是未应变面的两条边。 注意 Ei 是相对于由形因子定义的原始边定义的， 因此由您确保一切正常工作。仔细测试！！！ elastic_coeff 属性必须由用户创建和初始化。
元素：面。
参数：无。
模型：线性。
环境维度：3。
海森矩阵：是。
示例数据文件声明： 
   define facet attribute elastic_basis real[2][2]
   define facet attribute elastic_coeff real[6]
   quantity genlastic energy method general_linear_elastic global

relaxed_elastic、 relaxed_elastic1、relaxed_elastic2

命名方法。 描述： linear_elastic 方法的变体。 基于 Cauchy-Green 应变矩阵计算面的线性弹性应变能量， 压缩计为零能量，模拟例如塑料薄膜。 效果是允许起皱。 设 S 为未应变面的 Gram 矩阵（边的点积）。 设 Q 为 S 的逆矩阵。 设 F 为应变面的 Gram 矩阵。 设 C = (FQ-I)/2，Cauchy-Green 应变张量。 设 v 为泊松比。 则能量密度为 
   (1/2/(1+v))(Tr(C^2) + v*(Tr C)^2/(1-(dim-1)*v))
每个面都有额外属性 poisson_ratio 和 每个顶点有两个额外坐标，未应变表面在平面中的坐标。 因此表面必须设置为五维。 通过对应变张量进行特征值分析来检测压缩， 并丢弃任何负特征值。特征值可以由 relaxed_elastic1_A（较低特征值）和 relaxed_elastic2_A（较高特征值）方法分别访问，这些方法用于 info_only 模式。 还可以有实值面额外属性 LEBweight，可用于给 每个面的能量加权。 对于示例数据文件，参见 mylarcube.fe。 对于此方法的版本，使压缩零能量，参见 linear_elastic_B。
元素：面。
参数：无。
模型：线性。
环境维度：5。
海森矩阵：是。
示例数据文件声明： 
   space_dimension 5
   quantity lastic energy method relaxed_elastic global

carter_energy

命名方法。 描述： Craig Carter 的能量。 
给定 R^3 中的体 B_1 和 B_2，定义能量
    E = \int_{B_1}\int_{B_2} {1 \over |z_1 - z_2|^{p} } d^3 z_2 d^3 z_1
这简化为
E = {1\over (3-p)(2-p)}\sum_{F_2\in\partial B_2}\sum_{F_1\in\partial B_1}
    N_1 \cdot N_2 \int_{F_2}\int_{F_1}{1\over |z_1 - z_2|^{p-2}}
    d^2 z_1 d^2 z_2.
如果我们用质心 \bar z_1 和 \bar z_2 粗略近似，
E = {1\over (3-p)(2-p)}\sum_{F_2\in\partial B_2}\sum_{F_1\in\partial B_1}
        {A_1 \cdot A_2 \over |\bar z_1 - \bar z_2|^{p-2}},
其中 A_1 和 A_2 是面的非归一化面积向量。
幂 p 由变量 carter_power 设置（默认 6）。

元素：面。
参数：无。
模型：线性。
环境维度：3。
海森矩阵：否。
示例数据文件声明： 
   parameter carter_power  6     // 默认值
   quantity craig energy method carter_energy global

laplacian_mean_curvature

命名方法。 描述：将顶点的速度计算为表面平均曲率的 Laplacian， 用于模拟烧结中原子的表面扩散。每个顶点处的平均曲率计算为标量， 与面积归一化面积梯度相同，即面积梯度与体积梯度的点积，除以周围面的面积。 然后使用有限差分计算平均曲率的 Laplacian。这仅计算速度； 能量始终为 0。此方法仅应与 'g' 命令中的固定比例一起使用。

顶点的相对速度可以通过顶点属性 lmc_mobility 控制， 如果用户需要，应声明该属性。 如果用户想要访问方法找到的平均曲率值， 用户应定义顶点标量属性 lmc_mean_curvature。 此方法理想地保持体积，但由于数值不准确，您可能仍想添加体积约束。

警告：此方法仅应与固定的 'g' 比例因子一起使用。 为了稳定性，该因子应与最短边的四次方成正比， 因为平均曲率的 Laplacian 是四阶导数算子，类似于 0.001*length^4。 对于高度细化的表面，这可能导致非常缓慢的演化。


元素：顶点。
参数：无。
模型：线性字符串和线性肥皂膜。
环境维度：任意。
海森矩阵：否。
示例数据文件声明： 

  area_method_name "null_area"
  define facet attribute lmc_mobility real
  define facet attribute lmc_mean_curvature real
  quantity lmc energy method laplacian_mean_curvature global
