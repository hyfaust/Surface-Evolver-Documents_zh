几何元素

表面由各个维度的几何元素定义。每个元素都有自己的属性集。有些属性可以由用户设置；其他属性在内部设置，但可以由用户查询。还可以为任何类型的元素动态定义额外属性，这些属性可以是单个值或值向量。属性值可以在数据文件中指定，并通过命令查询。

元素：

  • 顶点
  • 边
  • 面
  • 体
  • 面边

顶点

顶点是空间中的一个点。 顶点的坐标是确定表面位置的参数。 当表面演化时，改变的是坐标。 顶点本身不携带默认能量，但可能通过位于 水平集约束上（在 弦模型中）， 或者通过应用命名量 能量而具有能量。 原始表面的顶点在数据文件的 顶点部分中定义。

边

边是一维 几何元素。 在线性模型中， 边是尾部顶点和头部顶点之间的有向线段。 在二次模型中，边由两个端点和一个中点的二次插值定义。 在拉格朗日模型中，边由边顶点的适当阶插值定义。 在弦模型中，边携带默认的表面张力能量，与其长度成正比。 边也可能通过位于 水平集约束上（在 皂膜模型中）， 或者通过应用 命名量能量而具有能量。 原始表面的边在数据文件的 边部分中定义。

面

在皂膜模型中， 面是由三个边循环定义的有向三角形。 在线性模型中， 面是一个平面三角形。 在二次 模型中，面是由三个面角顶点和三个边中点的二次插值定义的表面。在 拉格朗日模型中， lagrange_order 插值在 (lagrange_order+1)(lagrange_order+2)/2个顶点之间进行。 虽然单个面是有向的，但相邻面的方向没有限制。 默认情况下，面携带等于其面积的表面张力能量。

在弦模型中， 面是任意数量边的链。链不需要闭合。通常，在弦模型中定义面是为了定义体，因此空间维度为2，面是平面的，一个面对应一个体。 面本身不携带能量。

在单纯形模型中， 面是维度为 surface_dimension 的单纯形，由surface_dimension+1个顶点定义。surface_dimension 可以是小于或等于 space_dimension 的任何维度。 单纯形根据顶点的顺序定向。 默认情况下，单纯形携带与其体积成正比的表面张力能量。

面可以通过应用 命名量能量 携带额外能量。

原始表面的面在数据文件的 面部分中定义。

体

体是空间的全维区域。体不是三角化的。相反，它们由其边界 面（或在二维中为边）确定。 这些面用于计算体体积 和重力能量。只需要给出正确计算所需的那些面。在 弦 模型中，通常一个体对应一个面。 原始表面的体在数据文件的 体部分 中定义。

面边是面及其一条边的配对，其方向使得边的方向与面的方向一致。面边在Evolver内部使用，很少对用户感兴趣。它们不携带能量。 'C' 完整性检查命令有时会引用面边，如果表面不一致。Facetedge 可以用作元素生成器。

元素属性

下面是可能的元素属性列表。前几个适用于所有类型的元素。然后是专门适用于顶点、边、面和体的属性。参见 几何元素以获取每种类型元素的属性列表。

所有类型元素的属性

id

几何元素只读整数属性。 元素的id是与该元素唯一关联的正整数。Evolver将按照从数据文件读取的顺序为元素分配id，除非数据文件顶部有 -i命令行选项 或 keep_originals， 在这种情况下，数据文件元素编号就是id。 无论哪种情况，你都可以使用 original属性访问数据文件id。示例： 
   list vertex where id < 10
   set edge color red where id == 4 or id == 6 or id == 9
   foreach facet ff do
   { printf "%d  %d %d %d\n",ff.id,ff.edge[1].id,ff.edge[2].id,ff.edge[3].id }

oid

几何元素只读整数属性。 元素的oid是在表达式中使用的"有向id"。它是根据元素的使用方向与存储方向相同还是相反而签名的id编号。示例：要获取数据文件中面的边列表，使用oid而不是id： 
   foreach facet ff do { printf "%d  %d %d %d\n",ff.id,ff.edge[1].oid,
      ff.edge[2].oid,ff.edge[3].oid }

fixed

几何元素可读写布尔属性。 对于顶点，fixed 意味着它们在各种演化和三角化整理命令期间不会移动。对于边和面，fixed 意味着通过细化从它们生成的顶点是fixed，尽管声明面或边为fixed不会自动使其顶点为fixed。

对于体，fixed 意味着其体积被约束为其 target值。同样，作为命名量属性的fixed意味着该量值被约束。

固定性可以通过 fix和unfix命令更改。

示例： 

   fix vertex where z = 0
   unfix vertex where on_constraint 1
   list edge where fixed
   print body[1] fixed

on_constraint

顶点、 边或 面 只读属性。 布尔属性，用于判断元素是否在给定 约束上。 属性的完整语法是"on_constraint n" ，其中n是约束的编号或名称。示例： 
   list edge where on_constraint 3
   print vertex[3].on_constraint floorcon

on_boundary

顶点、 边或 面 只读布尔属性。 元素是否在 参数边界上的状态。 属性的完整语法是"on_boundary n"，其中 n是边界编号或名称。 示例： 
  list vertex where on_boundary 1
  list edge where on_boundary topbdry

on_quantity

顶点、 边或 面 只读属性。 布尔属性，用于判断元素是否对给定 命名量有贡献。 实际上，它测试元素是否在构成该量的任何方法实例上。 属性的完整语法是 "on_quantity quantityname"。 示例： 
   list facet where on_quantity  center_of_mass_x
   print vertex[3].on_quantity blue_area

on_method_instance

顶点、 边或 面 只读属性。 布尔属性，用于判断元素是否对给定 命名方法实例有贡献。 属性的完整语法是 "on_method_instance instancename"。 示例： 
   list facet where on_method_instance  center_of_mass_x_edges
   print vertex[3].on_method_instance blue_area_1

original

几何元素可读写整数属性。 对于从数据文件读取的元素，这是数据文件中赋予该元素的编号，可以通过定义元素的数据文件行中的显式original属性值覆盖。该值被细分产生的所有同类型元素继承。 对于在运行时以其他方式生成的元素，original属性值为-1。 示例：显示数据文件中哪些面由面1派生： 
   set facet color red where original == 1

命名量值作为属性

命名量和 方法实例可以应用于几何 元素，无论是在数据文件中（通过 在定义元素的行中添加量或方法名称）还是 使用set命令。 非全局量 或方法可以为单个元素unset。单个元素的值可以使用属性语法访问。 示例：假设有一个命名量"xmoment"，可以 为面求值。那么可以给出命令 
   foreach facet do printf "%d %f\n",id,xmoment
   list facet where xmoment > 4

额外属性

几何元素可读写属性。 如果额外属性 已在数据文件中或使用 define命令定义， 则可以使用属性语法访问。数据文件中的额外属性值可以通过在定义元素的行中添加属性名称和值来为元素初始化。额外属性也可以是数值数组 （全局数组可以是字符串，但属性还不能），使用标准嵌套括号语法初始化。示例： 
  define vertex attribute newx real
  define vertex attribute vmat real[3][2]
  vertices
  1   2 0 0 newx 3 vmat {{1,2},{3,4},{5,6}}
命令语言可以使用与内置属性相同的语法使用名称，并且可以在运行时定义额外属性： 
  set vertex newx x
  define edge attribute vibel real[2]
  set edge[2] vibel[1] 3; set edge[2] vibel[2] 4
  print vertex[3].newx
属性数组大小可以在运行时通过执行属性的另一个定义来更改，但维度数量必须相同。更改大小时，数组条目值会尽可能保留。

额外属性的值也可以由用户提供的代码计算。属性定义后跟关键字"function"，然后是括号中的代码。在代码中，关键字"self"用于引用正在计算属性的元素。 示例：将面的最低z值实现为属性： 

 define facet attribute minz real function
	 {self.minz := min(self.vertex,z);}

边属性

边id

参见id通用元素。

边oid

参见oid通用元素。

边original

参见original 通用元素。

边长度

边只读实数属性。边的长度。 示例： 
 histogram(edge where on_constraint 1, length)
 print edge[3].length

边密度或张力

边可读写实数属性。 "密度"和"张力"是同义词。 每单位边长的能量。在弦模型中默认为1，在皂膜模型中默认为0。 可以在边部分的 数据文件中修改张力， 方法是在定义边的行中添加"tension value"。 张力可以通过set命令修改。 示例： 
  set edge tension .5 where id < 10
  loghistogram(edge,density)

固定边

边可读写属性。 对于"固定"的边，由细化边创建的任何顶点或边将继承"固定"属性。 在数据文件中声明边为固定不会固定边上的顶点，从命令提示符固定边也不会固定任何顶点。 可以在边部分的 数据文件中声明边为固定， 方法是在定义边的行中添加fixed。 从命令提示符，可以使用fix和 unfix命令固定或取消固定边。 示例： 
  fix edge where on_constraint 1
  list edges where fixed
  set edge color red where fixed
  unfix edge[3]

边顶点

边只读属性。作为边顶点的 生成器。 在线性模型中， 这意味着尾部和头部顶点。在 二次模型中， 这意味着尾部、头部和中间顶点。 对于拉格朗日模型， 这意味着从尾部到头部的所有顶点，按顺序排列。 对于单纯形模型， 这意味着按存储顺序排列的顶点。 示例： 
   list edge[2].vertices
   list edge ee where ee.vertex[1].on_constraint 1

边midv

边只读属性。 在二次模型中，给出 边的中点顶点的id。示例： 
  print edge[23].midv

边面

边只读属性。 生成 附加到边的面，当有意义时按边周围顺序排列， 面方向与边方向一致。 示例： 
   list edge[2].facets
   foreach edge ee do print max(ee.facets,area)

边价

边只读整数属性。 边的价是与其相邻的面的数量。 示例： 
  list edges where valence == 1
  refine edge where valence != 2

裸边

边可读写布尔属性。 声明边为"裸边"表示边没有相邻的面（皂膜模型）。最好在 数据文件中声明， 方法是在定义边的行中添加关键字bare。 有助于避免警告消息。 裸边可用于在图形中显示线框、框架、轮廓、轴等。示例： 
  list edge where bare

边约束

边可读写属性。 边可以放在 水平集约束上。 对于这样的边， 由细化边生成的任何顶点和边将继承该约束。边可以在 边部分的 数据文件中设置约束， 方法是在定义边的行中列出约束编号，前面加上关键字constraint。 将边放在约束上不会将其现有顶点放在约束上。 在命令中，边的状态可以使用 "on_constraint" 属性读取。状态可以使用 set或unset 命令更改。 示例： 
  list edge where on_constraint 2
  set edge constraint 1 where id == 4 or id == 6
  unset edge constraint 3

边on_constraint

参见on_constraint 通用元素属性。

边e_constraint_list

此只读属性提供边所受约束的列表。e_constraint_list[1]是 列表中约束的数量，后跟约束的编号。请注意，对于 命名约束，使用内部指定的编号。

边边界

边可读写属性。 如果边在 参数边界上， 则从边生成的任何边和顶点 将继承该边界。默认情况下， 新顶点参数值通过从边的一端外推来计算。 这避免了插值参数值会产生的环绕问题。但是，如果 interp_bdry_param开关打开，则使用插值。 边是否在边界上的状态可以使用 布尔属性on_boundary查询。 边可以从边界取消设置， 也可以设置到边界上（但需要小心正确操作）。 示例： 
  list edges where on_boundary 1
  unset edges boundary 2

边on_boundary

参见on_boundary通用元素。

边e_boundary

边只读整数属性。 内部边属性，保存边所在边界的id编号。请注意，命名边界 有内部id编号，在此使用。

边颜色

边可读写属性。 颜色用于图形。 默认颜色是黑色。颜色可以在 数据文件中设置，或使用 set命令设置。 在geomview中，边颜色 将仅显示满足 show边条件的边，然后 它们将与geomview绘制的边竞争，除非您 在geomview窗口中使用"ae"关闭geomview的边绘制。 示例： 
  set edge color red where length > 1
  show edge where color != black

边edge_vector

边只读属性。 边向量的分量。edge_vector[1]是第一个分量， edge_vector[2]是第二个分量，依此类推。 示例： 
 print edge[3].edge_vector[1]
 list edge ee where ee.edge_vector[1] > 0
 histogram(edge, edge_vector[1])

边no_refine

边和面 可读写布尔属性。防止在细化操作中对 边或面进行细化。不影响约束细化。 示例： 
  set edge no_refine
  set edge no_refine on boundary 3
  refine edge where not no_refine

边no_transform

边和面 可读写布尔属性。 当边或面设置为on_transform时，表示 在对称群模式下，对称变换不会应用于此边或面。 对称组以基本域覆盖的外部区域开始。 边界效应是基本域外没有对称变换。 参见环面文档。 示例： 
  set edge no_transform on boundary 1
  set facet no_transform on boundary 1

边wrap

边可读写属性。 当使用对称群时， 边穿过基本域边界时， 边被标记为移动边头顶点到相对于尾顶点的正确位置的群元素。 该群元素的编号存储在边的wrap属性中。 wrap默认为0（单位元素）。在边部分的 数据文件中设置wrap，方法是在 定义边的行中列出群元素的编号，前面加上关键字wrap。 参见对称群。 在环面模型中， 值从1到2*n， 其中n是维度，负方向和正方向的穿越各一个。 请注意，边顶点值和表面外观可能不会立即更新。 示例： 
  set edge wrap 3 where id == 1
  set edge[2] wrap 3

边wrap_list

边可读写整数数组属性。 当使用对称群时， 此数组保存环绕列表： 长度为适当群元素编号+1的数组，每个元素是 边穿过基本域边界时从尾到头的穿越方向的群元素编号。 对于给定索引i，显示的元素是列表中的第i个元素。 非零值只有在基本域边缘有多个路径时才相关。 此属性也可以作为"wrap"访问。 示例： 
  set edge[2] wrap_list 3
  set edge wrap_list 5 where on_constraint 1

边show

边和面 只读布尔属性。是否在图形中显示。参见 show命令。
示例： 
  show facet where color == green

边orientation

边整数可读写属性。 反映边的"方向"（或符号），当边作为 面的边界的一部分时使用。 值是1或-1。当边在面的正边界上时，值为1； 在负边界上时，值为-1。 默认为1。 示例： 
  list facet[1].edges
  set edge orientation -1 where on_boundary 2
  set edge[3] orientation -1

边frontbody

边只读整数属性。 在弦模型中， 这是连接到边"前面"的体的id编号。 "前面"指的是与边正方向一致的方向。 通常是一个体的 面生成器的一部分，该面的 正方向与边方向一致。 如果该边没有相邻的体，值为0。 示例： 
 print edge[3].frontbody
 list edge where frontbody == 1

边backbody

边只读整数属性。 在弦模型中， 这是连接到边"后面"的体的id编号。 "后面"指的是与边负方向一致的方向。 通常是一个体的 面生成器的一部分，该面的 负方向与边方向一致。 如果该边没有相邻的体，值为0。 示例： 
 print edge[3].backbody

边dihedral

边只读实数属性。 边两个面法线之间的角度（弧度）。 对于裸边或只有一个面的边，角度为0。 示例： 
 histogram(edge where not bare, dihedral)
 list edge where dihedral > .1

边noncontent

边可读写布尔属性。 防止边对体内容（体积）产生贡献。 在环面模型中，边可能不是封闭体边界的一部分， 因此它们不应对其产生贡献。 示例： 
  set edge noncontent
  set edge noncontent on constraint 1
  set edge[2] noncontent

面密度或张力

面可读写实数属性。 "密度"和"张力"是同义词。 单位面积的能量密度。在 皂膜模型中默认为1，在 弦模型中默认为0。 可以在数据文件的 面属性中修改。 可以使用set命令修改。 张力可以是负的（对面积惩罚），这意味着面积应该最小化。 但是，如果在任何边或角上负张力面与零张力面相邻，则表面可能会爆炸，因为 负张力面为了最大化面积而趋于无穷。 在最小表面问题中，不需要负张力面； 只需将其他面的张力设置为0，它们就不会有 面积能量。 在弦模型中， 面与面之间的边界边张力由 相的facet_tension确定。在数据文件中给出相之间的张力表。 注意：在皂膜模型中，密度还用作乘以 拓扑类型3体积约束 时的常数。
示例： 
  set facet tension 0
  set facet tension 1 where color == white
  set facet tension 0.5 where on_boundary 1

面面积

面只读实数属性。 面的当前面积。 这是density乘以几何面积的乘积。 示例： 
 histogram(facet,area)
 print sum(facet,area)
 list facet where area > 1

固定面

面可读写属性。 对于"固定"的面，由面细化创建的任何边或顶点将继承"固定"属性。 可以在面部分的 数据文件中声明面为固定， 方法是在定义面的行中添加fixed。 可以从命令提示符使用fix和 unfix命令固定或取消固定面。
示例： 
  fix facet where on_boundary 2
  list facets where fixed

面顶点

面只读属性。 生成 面的顶点，按面周界顺序排列，以给定面方向一致的顺序。 示例： 
  list facet[2].vertices
  foreach facet ff where ff.vertex[1].on_constraint 1 do
    { refine ff; }

面边

面只读属性。 生成 面的边，按面周界顺序排列。边的方向与面方向一致。 示例： 
  list facet[2].edges
  list facet ff where sum(ff.edges,length) > 10

面体

面只读属性。 在皂膜模型中， 生成 与面相邻的体（最多两个）。 警告：体的顺序不确定。 对于单个相邻体，可能作为body[1]或body[2]出现。 建议使用frontbody和backbody代替。 示例： 
  list facet where bodies == 0
  list facet[2].bodies

面frontbody

面只读整数属性。 在皂膜模型中， 体的id编号，位于面"前面"。 "前面"是法线的方向。如果该面没有相邻体， 值为0。 示例： 
 list facet where frontbody == 1
 list facet where frontbody == 1 and backbody == 2
 print facet[3].frontbody

面backbody

面只读整数属性。 在皂膜模型中， 体的id编号，位于面"后面"。 "后面"是法线的相反方向。 如果该面没有相邻体，值为0。 示例： 
 list facet where backbody == 2
 list facet where frontbody == 1 and backbody == 2
 print facet[3].backbody

面f_body_list

面只读整数数组属性。 与面相邻的体的列表，第一个元素是数量。 示例： 
 print facet[5].f_body_list[1]
 print facet[5].f_body_list[2]

面价

面只读整数属性。 面的价是边的数量。 例： 
 list facets where valence > 3
 list facet where valence == 2

面约束

面可读写属性。 面可以放在 水平集约束上。 对于这样的面，细化面生成的任何边和顶点将继承该约束。 面可以在面属性的 数据文件中设置约束， 方法是在定义面的行中列出约束编号，前面加上关键字constraint。 将面放在约束上不会将其现有边和顶点放在约束上。 在命令中，面的状态可以使用 "on_constraint" 属性读取。状态可以使用 set或unset 命令更改。 示例： 
  list facet where on_constraint 2
  set facet constraint 1 where id == 4 or id == 6
  unset facet constraint 3

面on_constraint

参见on_constraint 通用元素属性。

面f_constraint_list

此只读属性提供面所受约束的列表。f_constraint_list[1]是 列表中约束的数量，后跟约束的编号。请注意，对于 命名约束，使用内部指定的编号。

面边界

面可读写属性。 如果面在参数边界上， 则从面生成的任何面、边和顶点将继承该边界。 默认情况下，新顶点参数值通过从面的一角外推来计算。 这避免了插值参数值会产生的环绕问题。但是，如果 interp_bdry_param开关打开，则使用插值。 面是否在边界上的状态可以使用 布尔属性on_boundary 查询。面可以从边界取消设置， 也可以设置到边界上（但需要小心正确操作）。 示例： 
  list facets where on_boundary 1
  unset facet boundary 2

面on_boundary

参见on_boundary通用元素。

面f_boundary

面只读整数属性。 内部面属性，保存面所在边界的id编号。请注意，命名边界 有内部id编号，在此使用。

面颜色

面可读写属性。 颜色用于图形。 可以在数据文件中设置，或使用 set命令设置。与 color words兼容的任何字符串都可以用作颜色属性。 默认颜色是透明的。 "透明"面不显示，因此需要将其颜色更改为可见的东西。 显示条件也在 show命令中可以指定。 示例： 
  set facet color cyan
  set facet color red where id > 20
  show facet where color == cyan

面frontcolor

面可读写属性。 面"前面"的颜色，即法线方向。 默认使用面的一般颜色。 可以与backcolor结合使用以获得双面颜色。 示例： 
  set facet frontcolor white
  set facet backcolor red

面backcolor

面可读写属性。 面"后面"的颜色，即法线的相反方向。 默认使用面的一般颜色。 可以与frontcolor结合使用以获得双面颜色。 示例： 
  set facet backcolor blue
  set facet frontcolor red

面opacity

面可读写属性。 面的不透明度。值为0表示完全透明，1表示完全不透明。 默认为1。用于部分透明的彩色图形。 实际上，这是一个值， 在0到1之间，其中0是完全透明的，1是不透明的。 示例： 
  set facet opacity .4

面facet_normal

面只读属性。 面法线的分量。facet_normal[1]是第一个分量， facet_normal[2]是第二个分量，依此类推。 在二次模型中， 法线基于高斯积分，因此对于低张力面可能不准确。 如果您需要准确的法线，可以使用convert_to_quadratic 生成高阶模型中的法线，使用convert_to_lagrange。 示例： 
  print facet[3].facet_normal[1]

面no_display

面可读写布尔属性。 防止面在图形中显示。 与使用show命令不同， 面在每次显示更新后不会丢失此标志。 但是，仍然使用show命令来实际显示东西； no_display标志只是一个额外的过滤器。 示例： 
  set facet no_display
  set facet no_display on boundary 1
  unset facet no_display where id < 10

面no_refine

参见no_refine边属性。

面no_transform

参见no_transform 边属性。

面orientation

面整数可读写属性。 反映面的"方向"（或符号）。 值是1或-1。正方向是通常显示的面法线方向。 示例： 
  list facet where orientation != 1
  set facet orientation -1 where id == 3

面noncontent

面可读写布尔属性。 防止面对体内容（体积）产生贡献。 在环面模型中， 面可能不是封闭体边界的一部分， 因此它们不应对其产生贡献。 示例： 
  set facet noncontent
  set facet noncontent on constraint 1
  set facet noncontent where id > 10
  unset facet[5] noncontent

面phase

面可读写整数属性。 当有phasefile时， 此属性决定皂膜模型中两个体之间面的张力。 此属性还有一个弦模型的 类比：phase边属性。 另见弦模型。 示例： 
   list facet where phase == 1

面f_method_list

面只读整数数组属性。 面拥有的内部方法实例列表。 f_method_list[1]是列表中方法实例的数量，后跟 方法实例的编号。请注意，这是一个内部列表，方法实例 有内部编号，在此使用。

面f_next_bfacet

体属性

体id

参见id通用元素。

体oid

参见oid通用元素。

体original

参见original 通用元素。

体面

体只读属性。 生成 体的面。 示例： 
  list body[2].facets
  list body bb where sum(bb.facets,area) > 10

体密度

体可读写实数属性。 每单位体积的能量密度。默认为1。 可以在体部分的 数据文件中设置密度。 可以使用set命令修改。 密度是体的自适应能量。对 Hessian迭代没有影响。 但是，对于非内容 体积约束，密度用作 体积约束的乘数（必须非负）。 示例： 
  set body density 2
  set body density 0 where id == 1

体体积

体只读实数属性。 体的当前体积。 在环面模型中， 此值通过环面方法计算。 请注意，在环面模型中，如果体在基本域中有任何 三角形，则基于解纠缠三角形计算体积， 因此体积可能与您从纠缠网格看到的明显不同。 示例： 
 list body where target > volume
 list body bb where bb.target > bb.volume
 histogram(body,volume)

体target

体可读写实数属性。 体积约束 的目标体积。可以使用 set命令设置，也可以在 体部分的 数据文件中设置。 在数据文件中，target是默认关键字，但如果需要， 可以使用单词"target"。请参见下面的示例。 请注意，target必须与实际体积和 volconst的符号相同。如果约束的体积为0，则必须 用正的target和volconst创建，因为实际体积（至少对于自相交的 环面网格）可能在符号上振荡。 示例： 
  set body target 10 where id == 1
  set body target volume
  set body target 0 where target == 0

体volfixed

体可读写布尔属性。 固定的 体不会在平衡 或移动操作期间的迭代中 调整其体积。 这可能有用， 例如，当您想根据对称性约束一个体，但不 调整其他体时。 示例： 
  set body volfixed where id == 1
  set body volfixed where target == 0
  unset body volfixed where id == 2

体volconst

体可读写实数属性。 体积约束 的体积常数。添加到计算的 几何体积以获得用于约束的总体积。 用于在细化时跟踪约束中丢失的少量体积。 在环面模型中， 用于跟踪 环面方法贡献。 请注意，volconst使用内部符号约定， 与target相反。 示例： 
  set body volconst 0 where id == 1
  print body[1].volconst
  print body[1].target - body[1].volume - body[1].volconst

体actual_volume

体只读实数属性。 纯几何体积，不考虑volconst。 此属性是只读的。 示例： 
  print body[1].actual_volume
  print body[1].actual_volume + body[1].volconst

体pressure

体只读实数属性。 体的当前压力。压力是体积约束的 拉格朗日乘子。 在eigenprobe和 linear命令的输出中报告。 可以使用set命令显式设置压力。 参见表达式中的 "pressure"。 示例： 
  set body pressure 0
  print body[1].pressure
  print pressure_of_body 1

体phase

体可读写整数属性。 当有phasefile时， 此属性确定体的相。 此属性还有一个弦模型的类比： phase边属性。 另见弦模型。 示例： 
   list body where phase == 1

体centerofmass

体可读写布尔属性。 布尔体属性。适用于环面模型 中图形显示的"连接"体模式。当为此设置一个体时， 体显示的质心被记住，下次绘制体时， 其环绕使得其新质心接近其先前质心。 这防止了靠近基本域边界的体在演化过程中 轻微移动时来回跳跃。默认开启。 示例： 
 set bodies centerofmass

体量

目前没有为体实现命名方法，因此命名量和方法不适用。
