约束和边界

Surface Evolver 的常规操作模式是在约束条件下最小化能量。 约束有两大类：

  • 逐点约束：
    • 固定顶点
    • 水平集约束
    • 单侧约束
    • 参数化"边界"曲线和表面
  • 全局约束：
    • 体积约束
    • 命名量约束

水平集约束

水平集约束是将顶点限制在函数的零水平集上。 公式可以包含任何 Evolver 已知值的表达式，给定特定顶点。 最常见的是只使用顶点的坐标 (x,y,z)，但也可以使用变量、量值或顶点额外属性。 使用顶点额外属性是将单个公式自定义到各个顶点的好方法。 例如，如果有一个名为 zfix 的顶点额外属性，可以通过一个公式为 z = zfix 的约束将顶点强制到各自的 z 值，当然要先为每个顶点分配适当的 zfix 值。 水平集约束可以有几种角色：

  • 顶点可能需要位于约束上（等式约束）或一侧（不等式约束）。 约束可以声明为 GLOBAL， 此时它适用于所有顶点。 参见 mound.fe 示例。
  • 约束可以有一个相关的 能量向量场， 该向量场在位于约束中的边上积分以给出 能量。 这对于指定壁面接触角和计算重力势能很有用。 不对 FIXED 的边进行积分。参见 mound.fe 示例。 在 弦模型 中， 能量被积函数是在约束上的顶点处计算的单个分量。
  • 约束可以有一个相关的 内容向量场， 该向量场在位于约束中的边上积分以给出对体的 体积 贡献， 该体的边界面包含这些边。这对于在不完全用其他无用面片包围体的情况下获得正确的体积很有用。 理解内容如何添加到体中对于正确设置符号非常重要。 积分沿边的正方向进行。如果边在面上是正定向的，且面在体上是正定向的，则积分添加到体中。 这可能最终给出与你认为的自然方向相反的被积函数符号。 不对 FIXED 的边进行积分。参见 tankex.fe 示例。 在 弦模型 中， 内容被积函数是在约束上的顶点处计算的单个分量。
  • 有一个约束属性 content_rank，如果顶点（弦模型）或边（肥皂膜模型）位于多个具有内容积分的边界上（例如墙壁相交处），则如果存在内容等级，等级最低的内容积分将贡献给负侧体的内容，最高等级的内容将贡献给正侧体的内容。
  • 约束可以声明为 CONVEX， 此时约束中的边具有与其相关的能量，该能量与直边和表面壁之间的面积成正比。 这个能量（称为"间隙能量"）旨在补偿平面面片遇到表面壁时通过延长壁上的一些边并缩短其他边来最小化其面积的趋势，其净效果是增加边与壁之间的净间隙。参见 tankex.fe 示例。

水平集约束在 数据文件 的顶部声明。 它们可以应用于 顶点、 边 或 面片。 约束通常应用于顶点和边，如 mound.fe。 请记住，你需要将约束应用于边，以便该约束应用于通过细化在该边上创建的顶点。 有时将约束应用于面片，通常是为了使面片符合预定形状。 确保应用于顶点的约束在顶点处线性无关。

约束通常在数据文件的 顶点、 边 和 面片 部分中应用， 但也可以通过 set 或 unset 命令设置或移除。示例： 

set vertex[4] constraint 4
unset edge constraint 1 where id < 10
取消设置不在约束上的元素没有问题。 当顶点设置到约束上时，顶点坐标会立即投影到约束上。 将边设置到约束上不会设置其顶点。面片也是如此。

几个元素属性在水平集约束中很有用：

  • on_constraint - 布尔属性，表示元素是否已被放置在约束上。
  • hit_constraint - 布尔顶点属性，表示约束是否精确满足约束。 用于单侧约束；对于精确约束始终为 1。
  • value_of_constraint - 顶点属性，是约束公式在顶点处的值。
  • v_constraint_list、- e_constraint_list、- f_constraint_list-- 向量属性，其第一个元素是元素所在的约束数量，后跟约束编号列表（使用命名约束的内部编号）。
  • 约束法线 - 在顶点处垂直于水平集的单位向量。
特定水平集约束是等式约束还是单侧约束可以在运行时通过表达式查询： 
     is_constraint[number].fixed
或   is_constraint[name].fixed
"number" 可以是表达式；"name" 是约束的不带引号的名称（如果有的话）。 如果约束是等式约束，则值为 1，否则值为 0。列出顶点所在的所有等式约束的示例： 
foreach vertex vv do
{ for ( inx := 1 ; inx < vv.v_constraint_list[1] ; inx++ )
    if is_constraint[vv.v_constraint_list[i+1]].fixed then
      printf "Vertex %d on equality constraint %d\n",
        vv.id,vv.v_constraint_list[i+1];
};

单侧约束

如果 水平集约束 在 数据文件 中声明为 NONNEGATIVE 或 NONPOSITIVE， 受约束影响的顶点必须保持在水平集函数域的该部分。 通常不建议为单侧约束上的边提供边积分或声明它们为 CONVEX。 顶点是否精确满足约束可以通过顶点的 hit_constraint 属性查询。 'g' 迭代步骤会检查顶点是否想要离开它已经遇到的单侧约束，但 hessian 命令不会；因此，当涉及单侧约束时，明智的做法是在 hessian 或 hessian_seek 之间穿插 'g'。

示例：假设要将气泡保持在半径为 5 的球形罐内。那么可以在数据文件中定义约束： 

constraint 1 nonpositive
formula: x^2 + y^2 + z^2 = 25
为了评估非负性或非正性，所有项都移到公式的左侧。 然后将此约束应用于气泡表面的所有顶点、边和面片。

如果定义实值顶点额外属性 one_sided_lagrange，则将记录遇到单侧约束的顶点的拉格朗日乘子。 one_sided_lagrange 可以定义为数组。如果顶点遇到的约束多于 one_sided_lagrange 的大小，则只记录前几个合适的。

约束的类型可以在运行时作为表达式查询：

is_constraint[number].nonnegative
is_constraint[name].nonnegative
is_constraint[number].nonpositive
is_constraint[name].nonpositive
is_constraint[number].fixed
is_constraint[name].fixed
如果约束是指定类型，则值为 1，否则值为 0。 "number" 可以是表达式；"name" 是约束的不带引号的名称（如果有的话）。 示例： 
print is_constraint[floorcon].nonnegative

参数化"边界"曲线和表面

顶点位置可以用参数化曲线或表面上的参数给出。 在 Evolver 术语中，这样的曲线或表面称为"边界"，因为它们通常用作表面的边界曲线， 例如，金属丝环上的肥皂膜可以将金属丝实现为边界。 顶点、边和面片可以被视为位于边界中。对于顶点，这意味着顶点的基本参数是边界的参数， 其坐标根据这些参数计算。边界上的顶点在迭代期间可以移动，除非声明为固定。 参见 cat.fe 示例。

边界在 数据文件 的顶部定义。 边界上的顶点在数据文件中列出时使用其参数值而不是坐标，并在每个这样的 顶点定义 后附加 "boundary n"。 边界上的边和面照常定义，但在每个定义后附加 "boundary n"。 因此数据文件中有如下行： 

boundary 1 parameters 1
x1:  cos(p1)
x2:  sin(p1)
x3:  0.75
...
Vertices
1   0.0  boundary 1
2   pi/3 boundary 1
...
Edges
1   1 2 boundary 1
...

将边放在边界上意味着在该边上创建的顶点将位于边界上。 边界上的边必须至少有一个端点位于边界上，用于外推任何创建顶点的边界参数。 外推而不是插值中点参数解决了在圆形或圆柱体等边界上的环绕问题。 但是，如果你确实想要插值，可以在数据文件顶部使用关键字 INTERP_BDRY_PARAM， 或使用切换命令 interp_bdry_param。 插值要求边的两个端点位于同一边界上，这在不同边界上的边相交时无法实现。 要处理这种情况，可以通过声明两个特定的顶点额外属性 extra_boundary 和 extra_boundary_param 向顶点添加额外边界信息： 

interp_bdry_param
define vertex attribute extra_boundary integer
define vertex attribute extra_boundary_param real[1]
然后在关键顶点上声明属性值，例如： 
vertices
1    0.00  boundary 1   fixed extra_boundary 2 extra_boundary_param 2*pi
如果在需要时顶点上未设置 extra_boundary 属性，Evolver 将悄悄地回退到插值。

将面放在边界上意味着从细化该面创建的所有边和顶点都将位于边界上。 在这种情况下，边界应该有两个参数（或表面的任何维度）。 这对于使表面符合已知参数形状很有用。

边界上的边具有类似水平集约束边的能量和内容积分，但它们在内部实现为 命名量。

元素是否位于特定边界上可以通过 on_boundary 布尔属性查询。 元素可以通过 unset 命令从边界中移除， 也可以通过 set 命令设置到边界上。 unset 的典型用途是使用 2 参数边界定义初始表面，细化几次，然后取消设置。示例： 

list vertex where on_boundary 2
unset vertex boundary 1 where on_boundary 1
unset edge boundary 1
unset facet boundary 1
set vertex[10] boundary 1
取消设置不在边界上的元素没有问题。

元素所在的边界编号在只读属性 __v_boundary、__e_boundary 或 __f_boundary 中。 如果元素不在边界上，则为 0。如果没有定义边界，则这些属性不存在。 请注意，命名边界在内部被分配编号，这些编号在这里显示。

顶点参数可以在表达式中作为属性 p1（以及 p2,... 用于更多参数）访问。 顶点参数可以通过 set 命令更改。示例： 

print vertex[5].p1
set vertex p1 p1+.1 where id < 4
vertex[2].p1 := 3
只要某个顶点在边界上（以便在顶点结构中为参数分配空间），访问不在边界上的顶点的参数不是错误。

一般准则是将约束用于二维墙壁，边界用于一维金属丝。 如果你使用边界金属丝，你可能可以将边界上的顶点和边声明为 FIXED。 然后边界就变成了细化边界边的指南。

注意：边界上的顶点不能同时具有约束。
