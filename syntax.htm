词法格式

对于了解这方面的人来说， 数据文件和命令是通过 lex 程序生成的词法分析器读取的。 规范在 datafile.lex 中。命令进一步由 yacc 生成的解析器解析。

在解析表达式时，使用最长的合法表达式。 这允许坐标通过几个连续的表达式指定，无需特殊分隔符。

注释

注释可以包含在 /* */ 对中（如 C 语言）， 可以跨越多行。// 表示该行的其余部分是注释， 如 C++ 中一样。

行和行拼接

文件由行组成。换行是有意义的。 下一个物理行可以通过在当前行末尾放置 \ 来拼接到当前行。 行拼接在 // 注释中无效。 空行和注释行可以自由放置在任何地方。 各种计算机系统使用的 CR 和 NL 的各种组合都能被识别。

大小写

在数据文件中，大小写不敏感，除了 #define 宏名称。 在运行时命令中，大小写仅对 单字母命令敏感。

空白字符

在数据文件中，空白字符包括空格、制表符、逗号、冒号和分号。 所以如果你想用逗号分隔坐标值，用冒号美化约束定义，都是可以的。 在命令中，空白字符包括空格和制表符。 CTRL-Z 也是空白字符，这是为了方便从 DOS 导入的文件。

标识符

标识符遵循标准 C 规则（由字母数字字符和 '_' 组成， 首字符不能是数字），且不能是关键字。 标识符用于宏名称（在数据文件中）和用户定义的 变量以及 命令。 标识符必须至少有两个字符， 因为单个字符可能与命令混淆。 要交互式地查找某个名称是否已作为关键字或用户定义名称使用， 请使用 help 命令。在脚本中， 可以使用 is_defined 函数，其语法为 
 is_defined(stringexpr)
stringexpr 必须是带引号的字符串或其他字符串表达式。 如果名称未定义则返回值为 0，已定义则为 1。 此函数在运行时求值，但整个命令中的变量在命令执行前就被解析， 所以像 if is_defined("newvar") then newvar := 1 else newvar := 2 这样的命令会给 newvar 赋值 1，即使这是它第一次出现。 在脚本中更好的测试方法是使用 define 命令来定义变量而不初始化，然后测试它是否有默认值， 即数字变量为 0，字符串变量的 sizeof 为 0。

数字

常量值可以是任何常用形式。这包括整数、定点数和科学计数法，例如 
      2    -3    .5    23.    5e-10    +0.7D2
以 0x 开头的十六进制整数也被接受，如 0x12Af， 以及二进制数如 11001b，由尾部的 'b' 表示。 颜色名称被解释为整数。

边 和 面的颜色 以整数形式记录在 -1 到 15 的范围内。 这些整数如何转换为屏幕上的颜色取决于 Evolver 的图形驱动程序的编写方式。 提供了以下同义词，并希望图形驱动程序能正确显示它们：

常量表达式

常量表达式在解析时求值。它们在各种语法定义中用 constexpr 表示。 它们主要出现在数据文件中。虽然它们可以包含变量， 但在解析后更改变量值不会产生影响。 变量表达式（在语法定义中用 expr 表示） 被记录为解析树，并在每次需要时重新求值。

x1,x2,...; x,y,z,w

根据上下文，空间坐标、顶点坐标、 边向量分量或面法向分量。

p1,p2,...

顶点在 参数化边界上的参数。

内部变量

toggle 名称

任何 toggle 命令 名称都可以用作表达式中的布尔变量（完整的 toggle 单词，不是单个字母）。 但要注意将 toggle 解释为命令或值时的歧义。 你可能需要强制将 toggle 解释为值。 "ad := autodisplay" 将 ad 设置为 autodisplay 的命令同义词， 而 "ad := (autodisplay)" 记录当前的布尔值。

字符串表达式

字符串表达式求值为一个字符串。目前， 产生字符串的唯一方式是：
  • 双引号字符串字面量，例如 "this is a string"。 识别以下标准 C 转义序列：
    • \n 换行
    • \r 回车
    • \t 制表符
    • \b 退格
    • \q 双引号
    • \c 字符 c 本身
    在 DOS、MS-Windows 或 Windows NT 路径中，使用 / 作为目录分隔符， 因为 \ 是转义字符。DOS 和 Windows 一直接受 / 作为目录分隔符。
  • 连续的双引号字符串在读取时会被连接成一个字符串。
  • 字符串变量，可以是内部的如 datafilename，或用户定义的。
  • sprintf 的输出。

算术运算符

这些是可以出现在表达式中的算术运算符：
+,-,*,/

普通的实数算术；+,-,* 也适用于数组。 注意：前面有空白字符、 后面跟数字的 '+' 或 '-' 被视为带符号数字。 因此 "3 - 5" 和 "3-5" 是单个表达式， 但 "3 -5" 不是。这是为了方便在数据文件中同一行上 列出多个表达式时分隔顶点坐标、度量分量等。

idiv

整数除法。先向零取整，然后进行整数除法。 例如：7 idiv 2 是 3；-3.5 idiv 2.1 是 -1；-3 idiv 2 是 -1。

%, mod

实数算术取模，x % y = x - floor(x/y)*y。

imod

整数算术取模， x imod y = floor(x) - floor(floor(x)/floor(y))*floor(y)。

=

不是赋值运算符。被视为低优先级的 '-'， 所以像 x^2 + y^2 - R^2 这样的水平集约束公式可以自然地写为 x^2 + y^2 = R^2。

(,)

用于分组和函数记法的括号。

^,**

求实数幂。

? :

条件表达式，如 C 语言中一样。 x ? y : z 在 x 非零时求值为 y，在 x 为零时求值为 z。

++,--

C 风格的前缀和后缀递增和递减命令。这些仅适用于 单个变量，并且只能作为独立命令使用，例如在 "for" 循环中： 

           for ( inx := 1 ; inx <= 10 ; inx++ )

布尔表达式 是算术表达式的子类， 零为假，非零为真；真结果求值为 1。

NOT, ! 布尔非
AND, && 布尔与，带短路求值。

matrix_inverse(A,B)

IS_DEFINED

要查明某个名称是否已作为关键字或用户定义名称使用， 请使用 is_defined 函数，其语法为 
 IS_DEFINED(stringexpr)
stringexpr 必须是带引号的字符串或其他字符串表达式。 如果名称未定义则返回值为 0，已定义则为 1。 此函数在运行时求值，但整个命令中的变量在命令执行前就被解析， 所以像 if is_defined("newvar") then newvar := 1 else newvar := 2 这样的命令会给 newvar 赋值 1，即使这是它第一次出现。 在脚本中更好的测试方法是使用 define 命令来定义变量而不初始化，然后测试它是否有默认值， 即数字变量为 0，字符串变量的 sizeof 为 0。

SIZEOF

返回数组或数组额外属性中的条目数。 也可以应用于字符串或字符串变量以获取字符串中的字符数。 语法： 
  SIZEOF(name)
  SIZEOF(string)
第一种形式中，name 是数组或额外属性的名称，不带引号。

VALID_BOUNDARY

布尔函数。根据具有给定编号的参数化边界是否存在返回 1 或 0 （注意，命名边界的名称在内部被解释为数字）。语法： 
 VALID_BOUNDARY(expression)
一个用途是与 high_boundary 内部变量结合使用来遍历所有参数化边界。例如， 
  for ( bnum := 1 ; bnum <= high_boundary ; bnum += 1 )
    if valid_boundary(bnum) then
       printf "Boundary %d has %d vertices on it\n",bnum,
          sum(vertex,on_boundary bnum);

VALID_CONSTRAINT

布尔函数。根据具有给定编号的水平集约束是否存在返回 1 或 0 （注意，命名约束的名称在内部被解释为数字）。语法： 
 VALID_CONSTRAINT(expression)
一个用途是与 high_constraint 内部变量结合使用来遍历所有水平集约束。例如， 
  for ( cnum := 1 ; cnum <= high_constraint ; cnum += 1 )
    if valid_constraint(cnum) then
       printf "Constraint %d has %d vertices on it\n",cnum,
          sum(vertex,on_constraint cnum);

VALID_ELEMENT

布尔函数。 根据具有给定索引的元素是否存在返回 1 或 0。语法： 
 VALID_ELEMENT(indexed_element)
示例： 
   if valid_element(edge[12]) then refine edge[12]
   if valid_element(body[2]) then set body[2].facet color red

用户定义内置函数

用户定义函数可以在 userfunc.c 中用 C 定义， 所以如果你想使用这些函数，需要自己编译 Evolver。 它们适用于表达式解释太慢的情况， 或需要椭圆积分等函数的情况。目前，它们自动是坐标的函数。 不要在表达式中给出任何参数； 例如 "(usr1 + usr3)/usr10"。

聚合函数

可以使用聚合函数对一组元素执行表达式的最大值、最小值、总和或平均值。语法为 
   aggregate(generator,expr)
其中 aggregate 是 max, min, sum, 或 avg， generator 是元素生成器。 示例：打印所有绿色面的总面积：
   print sum(facet where color == green, area)
"Count" 是一个聚合函数，它给出生成的元素数量，与表达式无关。

表达式中的元素属性值

任何元素属性的值都可以在表达式中使用。 如果存在默认活动元素，可以单独使用属性名。示例：
   foreach facet do print area
否则，属性被称为元素的字段： 
   foreach facet ff do print ff.area
索引额外属性需要方括号中的索引： 
   foreach facet ff do print ff.something[3]
索引值向下取整为整数值。索引从 1 开始。

大多数属性是数值或布尔类型，但有些是 元素生成器。

变量

Evolver 命令语言有自己版本的用户定义变量， 这是大多数编程语言都有的功能。变量根据赋给它们的值的类型进行分类： 数值型或字符串型。用户可以通过数据文件中的变量声明定义数值变量， 也可以通过在命令中赋值给标识符来定义两种类型。 变量可以通过在数据文件中声明为优化参数来进行优化。

数组

可以使用以下语法定义整数或实数的多维数组： 
   DEFINE variablename  REAL|INTEGER|STRING expr]...
此语法在数据文件头部和命令提示符下都有效。 如果数组已经存在，它将被调整大小，旧元素会尽可能保留。 不要用不同数量的维度来调整大小。注意数组索引从 1 开始。 大小为 0 是合法的，当你暂时不使用数组并想要释放存储空间时很有用。 有运行时数组边界检查。示例： 
   define fvalues integer[10][4]
   define basecoord real[10][space_dimension]
   define primcol string[3]
声明为 local 的标识符名称可以在过程和函数的数组声明中使用； 可以使用动态大小，但静态大小可能稍快一些，因为不需要动态分配内存。

在数据文件顶部，可以在定义后使用嵌套括号初始化器来初始化数组。例如： 

   define qwerty integer[4] = { 23, 45, 12, 2 }
   define vmat real[3][2] = {{1,2},{3,4},{5,6}}
   define primcol string[3] = {"red","green","blue"}
初始化器不需要完整；缺失的值将为零。

数组元素可以像编程语言中通常的那样进行赋值和引用： 

   fvalues[3][4] := 234;
   if basecoord[3][1] > 2 then print "hello world!\n";

数组初始化语法也适用于运行时对数组和数组切片的赋值， 包括作为数组的元素属性。初始化器中的条目必须是单个数字，不是数组。 赋值左右两侧的维度数必须一致，但每个维度的大小不需要一致。 右侧缺失的元素被视为零。示例： 

   vertex[1].__x := { 1.2, 3.1, 4.7 }
   define mat real[2][2];
   mat := {{1,2},{3,4}}
右侧在每次执行赋值时求值，因此右侧的条目可以是任何求数字的表达式。 += 和 -= 的赋值也有效，*= 和 /= 也有效，但请注意所有这些都按元素操作 （即 *= 和 /= 不是矩阵乘法和除法）。

print 命令可用于以括号形式打印整个数组或数组切片。示例： 

   print fvalues
   print fvalues[4]
有一些基本的全数组操作允许数组出现在赋值语句的左侧： 
   array := scalar
   array := array
   array := scalar * array
   array := array + array
   array := array - array
   array := array * array
   scalar := array * array  // 如果两个右侧数组都是一维的，则为点积
这里赋值两侧的 "array" 表示单个完整的数组； 不是产生数组的表达式或数组切片。 但 "scalar" 可以是任何求单个值的表达式。 对于乘法，数组必须是二维的且大小适当匹配。 这些操作也适用于作为数组的元素属性。

还有一个 matrix_inverse 过程和一个 matrix_determinant 函数。

对于一维数组，* 运算符给出对应条目乘积的总和。示例： 

   print vertex[1].__velocity * facet[3].__facet_normal

内部只读变量。体的数量。

breakflag

内部读写变量。 当设置为非零值时，导致命令解释器 中止并返回命令提示符。相当于 按键盘中断（通常是 CTRL-C）。中断 不会立即发生，而是在解释器循环中 定期检查用户中断的某个时刻。 用于从嵌套命令中退出，因为 return 只跳出当前过程。

body_dissolve_count

内部只读变量。被 dissolve 命令溶解的体的数量。 在命令执行结束时打印并重置为 0，或在执行 flush_counts 时重置。 也被 reset_counts 重置。

内部只读变量。PostScript 输出和屏幕图形中使用的中值灰度级。

内部只读变量。 最近一次 C 命令发现的错误数量。

clock

内部只读变量。Evolver 执行的总经过时间（秒）。读取系统进程经过时间，其分辨率通常相当粗糙，为 0.01 秒。对于纳秒计时， 请参阅 cpu_counter。

console_title

内部读写字符串变量。这是显示在 Evolver 命令控制台窗口上的标题。默认值是 "Surface Evolver - datafilename"（当然是当前数据文件的名称）。当你同时运行多个实例， 想要区分哪个在哪个控制台窗口中运行时很有用。只需将字符串赋给 console_title 就会自动更改窗口上的标题。

constraint_tolerance

内部读写变量。当顶点被投影到 水平集约束时， 通过牛顿法的投影重复进行 直到水平集函数小于 constraint_tolerance。 默认值 1e-12。

cpu_counter

内部只读变量。处理器周期计数器，仅在我知道如何访问的系统上可用（目前是 x86）。给出自系统启动以来的 CPU 周期数。请注意这是墙钟时间，不是进程时间。还要注意，当计算机休眠时它会重置为零，因此不保证在进程生命周期内单调递增！此外，机器中的多个处理器可能没有相同的 CPU 计数器，因此 Evolver 进程应分配给特定处理器（在 Windows 中，可以在任务管理器中通过设置进程的"亲和性"来完成）。 对于进程经过时间，请参阅 clock。

内部只读变量。包含当前数据文件名的字符串。
