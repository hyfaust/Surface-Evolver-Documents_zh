Surface Evolver 执行时间分析

粗略分析

粗略计时的分辨率约为 0.01 秒，可以使用 Evolver 的 clock 变量来完成。 例如，要对迭代进行计时，我通常的做法是： 
Enter command: quiet; start := clock; g 100; print start-clock; quiet off
这将以秒为单位打印出经过的时间。这里使用 quiet 来抑制屏幕输出， 因为屏幕输出会显著降低运行速度。

精细分析

纳秒级分辨率可以使用 Evolver 的 cpu_counter 变量来实现。

表达式求值分析

在 Evolver 具有 cpu_counter 功能且 Evolver 编译时定义了宏常量 PROF_EVALS 的系统上， Evolver 内部的表达式求值器会跟踪每次表达式求值期间所消耗的时钟周期数。 这些表达式包括过程（procedure）、函数（function）、约束（constraint） 和边界（boundary）公式、含量被积函数（content integrand）、 能量被积函数（energy integrand）、量被积函数（quantity integrand）等； 即所有在 dump 文件中以代码形式打印出来的内容。 print profiling 命令将打印出迄今为止每种表达式类型 所累积的 CPU 周期数。这些时间包含了所有子函数或子过程的执行时间。 以下是在 mound.fe 中运行 "gogo" 后的一个示例： 
Enter command: print profiling
Inclusive profiling counts:
                      Name            CPU Cycles
                              re           1,952,792
                            gogo         125,201,889
                           gogo2                   0
                           gogo3                   0
                           gogo4                   0
                           gogo5                   0

Constraint expressions
                    Constraint  Formula Cycles   Energy Cycles  Content Cycles
                             1         877,047       2,337,727               0
请注意，面积、体积等硬编码的计算不会显示在此处， 除非它们对整体经过时间产生影响。

reset_profiling 命令会将所有周期计数值重置为 0。
